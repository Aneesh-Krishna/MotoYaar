# Story 8.3 — Email Notifications for Document Expiry

**Epic:** 08 — Notifications & Background Jobs
**Story:** 8.3
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 8.1 (cron job calls emailService), Story 1.4 (Resend configured in src/lib/resend.ts)

## Story

**As a** user whose documents are about to expire,
**I want** to receive an email with the expiry details,
**so that** I get an alert even when I'm not actively using the app.

---

## Acceptance Criteria

1. - [ ] Warning email AND expiry-day email are sent by the cron job (Story 8.1 calls `emailService.sendDocumentExpiryEmail()`)
2. - [ ] Multiple expiring documents in the same cron run are combined into ONE email per user (not one per document)
3. - [ ] Email content includes: document type(s), vehicle name(s), expiry date(s), and CTA button "View in MotoYaar" linking to `/garage`
4. - [ ] `emailService.sendDocumentExpiryEmail(userId, docs)` in `src/services/emailService.ts`
5. - [ ] Uses `sendEmail(to, subject, html)` from `src/lib/resend.ts`
6. - [ ] HTML email template: simple, mobile-friendly, MotoYaar branded (orange `#F97316` accent)
7. - [ ] User can opt out: `users.email_notifications_enabled` field respected at cron run time — if `false`, email is NOT sent
8. - [ ] `email_notifications_enabled` column must exist on `users` table (default `true`); add migration if missing
9. - [ ] `pnpm typecheck` and `pnpm lint` pass

### emailService.sendDocumentExpiryEmail()

[Source: architecture.md#6-components]

Create `src/services/emailService.ts`:

```typescript
import { sendEmail } from "@/lib/resend";
import { db } from "@/db";
import { users } from "@/db/schema";
import { eq } from "drizzle-orm";
import { logger } from "@/lib/logger";

interface ExpiringDoc {
  type: string;           // e.g. "Insurance", "Registration", "Pollution Certificate"
  vehicleName?: string;   // from joined vehicle
  expiryDate: Date;
  daysUntilExpiry: number;
}

export const emailService = {
  async sendDocumentExpiryEmail(userId: string, docs: ExpiringDoc[]): Promise<void> {
    if (docs.length === 0) return;

    const user = await db.query.users.findFirst({ where: eq(users.id, userId) });
    if (!user) return;

    // Opt-out check
    if (user.emailNotificationsEnabled === false) {
      logger.info({ userId }, "Email notification skipped — user opted out");
      return;
    }

    const hasExpired = docs.some(d => d.daysUntilExpiry <= 0);
    const subject = hasExpired
      ? "Your document has expired — MotoYaar"
      : "Document expiry reminder — MotoYaar";

    const html = buildDocumentExpiryEmailHtml(user.name ?? "there", docs);

    try {
      await sendEmail(user.email, subject, html);
    } catch (err) {
      logger.warn({ err, userId }, "sendDocumentExpiryEmail failed");
      // Non-fatal — do not rethrow
    }
  },

  async sendBetaInviteEmail(email: string, inviteUrl: string): Promise<void> {
    // Story 11.5 implements this
    throw new Error("Not implemented");
  },

  async sendVehicleInviteEmail(
    inviteeEmail: string,
    vehicleName: string,
    inviteUrl: string
  ): Promise<void> {
    // Story 10.1 implements this
    throw new Error("Not implemented");
  },
};
```

### HTML Email Template

[Source: architecture.md#11-integrations]

`buildDocumentExpiryEmailHtml(userName, docs)` — mobile-friendly, branded:

```typescript
function buildDocumentExpiryEmailHtml(userName: string, docs: ExpiringDoc[]): string {
  const ctaUrl = "https://motoyaar.app/garage";

  const docRows = docs.map(doc => {
    const formattedDate = doc.expiryDate.toLocaleDateString("en-IN", {
      day: "numeric", month: "long", year: "numeric",
    });
    const statusLabel = doc.daysUntilExpiry <= 0
      ? '<span style="color:#dc2626;font-weight:600;">Expired</span>'
      : doc.daysUntilExpiry === 1
        ? '<span style="color:#f97316;font-weight:600;">Expires today</span>'
        : `<span style="color:#f97316;">Expires in ${doc.daysUntilExpiry} days</span>`;

    return `
      <tr>
        <td style="padding:8px 0; border-bottom:1px solid #f3f4f6; font-size:14px; color:#111827;">
          ${doc.type}${doc.vehicleName ? ` — ${doc.vehicleName}` : ""}
        </td>
        <td style="padding:8px 0; border-bottom:1px solid #f3f4f6; text-align:right; font-size:13px; white-space:nowrap;">
          ${formattedDate}<br/>${statusLabel}
        </td>
      </tr>`;
  }).join("");

  return `
    <!DOCTYPE html>
    <html>
    <body style="margin:0; padding:0; background:#f9fafb; font-family: -apple-system, sans-serif;">
      <div style="max-width:480px; margin:32px auto; background:white; border-radius:16px; overflow:hidden; box-shadow:0 1px 3px rgba(0,0,0,.1);">

        <!-- Header -->
        <div style="background:#F97316; padding:24px 32px;">
          <p style="margin:0; color:white; font-size:20px; font-weight:700; letter-spacing:-0.3px;">MotoYaar</p>
        </div>

        <!-- Body -->
        <div style="padding:32px;">
          <h2 style="margin:0 0 8px; color:#111827; font-size:18px;">Hi ${userName},</h2>
          <p style="color:#6b7280; font-size:14px; margin:0 0 24px;">
            ${docs.length === 1
              ? "One of your vehicle documents needs attention."
              : `${docs.length} of your vehicle documents need attention.`}
          </p>

          <table style="width:100%; border-collapse:collapse;">
            ${docRows}
          </table>

          <a href="${ctaUrl}"
             style="display:block; margin-top:24px; background:#F97316; color:white; text-align:center;
                    padding:14px 24px; border-radius:10px; text-decoration:none; font-weight:600; font-size:14px;">
            View in MotoYaar
          </a>
        </div>

        <!-- Footer -->
        <div style="padding:16px 32px; border-top:1px solid #f3f4f6;">
          <p style="color:#9ca3af; font-size:12px; margin:0; text-align:center;">
            You received this because you use MotoYaar. &nbsp;
            <a href="https://motoyaar.app/profile/settings" style="color:#9ca3af;">Manage notifications</a>
          </p>
        </div>

      </div>
    </body>
    </html>`;
}
```

### Opt-Out Enforcement

[Source: architecture.md#6-components]

The opt-out check is performed inside `emailService.sendDocumentExpiryEmail()` before calling `sendEmail()`. The cron job (Story 8.1) does not need to check opt-out separately — the service layer handles it.

The opt-out field `emailNotificationsEnabled` (camelCase Drizzle mapping of `email_notifications_enabled`) must be included in the Drizzle `users` schema definition.

### sendEmail() from resend.ts

[Source: architecture.md#11-integrations]

Read `src/lib/resend.ts` before calling. The function signature from architecture:

```typescript
sendEmail(to: string, subject: string, html: string): Promise<void>
```

If the function is a stub, implement it using the Resend SDK:

```typescript
// src/lib/resend.ts
import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendEmail(to: string, subject: string, html: string): Promise<void> {
  await resend.emails.send({
    from: "MotoYaar <notifications@motoyaar.app>",
    to,
    subject,
    html,
  });
}
```

The `from` address domain must be verified in the Resend dashboard. Use `RESEND_FROM_EMAIL` env var if configured.
