## Story

**As a** rider,
**I want** to earn badges for my riding milestones,
**so that** I have visible recognition for my journey and something to show off on my profile.

---

## Acceptance Criteria

1. - [ ] User profile page shows an earned badges section displaying all badges the user has unlocked
2. - [ ] Badges are calculated dynamically from existing user data on profile load — no manual granting required
3. - [ ] Initial badge set (Phase 1):

| Badge ID | Badge Name | Unlock Condition |
|----------|------------|-----------------|
| `first_vehicle` | First Garage | Added first vehicle |
| `first_trip` | First Journey | Logged first trip |
| `first_live_trip` | Gone Live | Completed first live GPS trip |
| `first_group_ride` | Pack Rider | Joined or hosted a multi-user live session |
| `km_500` | 500 km Rider | 500 km total across all tracked live trips |
| `km_1000` | 1,000 km Rider | 1,000 km total across all tracked live trips |
| `km_5000` | Road Warrior | 5,000 km total across all tracked live trips |
| `himalayan_rider` | Himalayan Rider | Completed a live trip with a route touching Ladakh / Himachal / Uttarakhand bounding box |
| `doc_streak` | All Clear | All vehicle documents valid (none expired or expiring) at time of check |
| `community_voice` | Community Voice | Posted 5 or more community posts |

4. - [ ] Unearned badges are displayed as greyed-out "locked" icons with the badge name visible — encourages users to pursue them
5. - [ ] Each earned badge shows its name and the date it was first earned
6. - [ ] Community posts optionally show the user's **highest-tier km badge** icon next to their username (e.g. `km_5000` shown if earned, otherwise `km_1000`, etc.)
7. - [ ] Badge evaluation runs on profile page load (server-side, cached for 1 hour) — no real-time push on unlock
8. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Badge Calculation Logic

All badge checks are pure DB queries — no separate badge table needed for MVP. A `user_badges` table can be added later for caching and "first earned on" dates.

```typescript
// src/services/badgeService.ts

export async function computeBadges(userId: string): Promise<Badge[]> {
  const [vehicles, trips, liveSessions, posts, routes] = await Promise.all([
    db.select({ count: count() }).from(vehicles).where(eq(vehicles.userId, userId)),
    db.select({ count: count() }).from(trips).where(eq(trips.userId, userId)),
    db.select({ count: count() }).from(liveSessionParticipants).where(eq(...)),
    db.select({ count: count() }).from(posts).where(eq(posts.userId, userId)),
    db.select({ distanceKm: tripRoutes.distanceKm }).from(tripRoutes)
      .innerJoin(trips, eq(trips.id, tripRoutes.tripId))
      .where(eq(trips.userId, userId)),
  ]);

  const totalKm = routes.reduce((s, r) => s + (r.distanceKm ?? 0), 0);

  return BADGE_DEFINITIONS.map(badge => ({
    ...badge,
    earned: badge.check({ vehicles, trips, liveSessions, posts, totalKm }),
  }));
}
```

### Himalayan Rider — Bounding Box Check

Approximate bounding boxes for qualifying regions:
```typescript
const HIMALAYAN_BOXES = [
  { name: 'Ladakh',      minLat: 32.5, maxLat: 36.0, minLng: 75.0, maxLng: 80.0 },
  { name: 'Himachal',   minLat: 30.5, maxLat: 33.5, minLng: 75.5, maxLng: 79.5 },
  { name: 'Uttarakhand', minLat: 28.5, maxLat: 31.5, minLng: 77.5, maxLng: 81.0 },
];

function routeTouchesHimalaya(waypoints: Waypoint[]): boolean {
  return waypoints.some(wp =>
    HIMALAYAN_BOXES.some(box =>
      wp.lat >= box.minLat && wp.lat <= box.maxLat &&
      wp.lng >= box.minLng && wp.lng <= box.maxLng
    )
  );
}
```

### user_badges Table (for "first earned" date)

```sql
CREATE TABLE user_badges (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  badge_id   TEXT NOT NULL,
  earned_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, badge_id)
);
```

On each profile load, newly computed earned badges are upserted into `user_badges` — this records the `earned_at` date without overwriting it on repeat loads.

### Caching

Wrap `computeBadges()` with `unstable_cache` (Next.js) with a 1-hour TTL:
```typescript
const getCachedBadges = unstable_cache(
  (userId) => computeBadges(userId),
  ['badges'],
  { revalidate: 3600 }
);
```
