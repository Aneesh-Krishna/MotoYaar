## Story

**As a** riding club organiser,
**I want** to create a club on MotoYaar and invite my members,
**so that** we have a shared space for our ride history, member list, and club feed instead of relying on WhatsApp groups.

---

## Acceptance Criteria

1. - [ ] Any logged-in user can create a **Club** with: name (required, unique), city (required), description (optional, max 300 chars), club logo/image (optional)
2. - [ ] The creator becomes the club **admin** (owner)
3. - [ ] Club has an **invite link** (`motoyaar.app/clubs/join/[code]`) — any logged-in user who opens the link is shown a join request prompt
4. - [ ] Joining a club requires **admin approval** (default) OR the admin can set the club to **open join** (anyone with the link is auto-approved)
5. - [ ] Club admin can: approve/reject join requests, remove members, promote a member to admin, change club settings
6. - [ ] The **Club Profile Page** shows: club name, city, logo, description, member count, member list (names + profile pics)
7. - [ ] The Club Profile Page has a **Club Feed** tab: posts shared specifically to the club (visible to members only); club members can post to the club feed using the existing post form with an added "Post to: [Club Name]" scope selector
8. - [ ] Club posts are visible **only to club members** — they do not appear in the main community feed
9. - [ ] The **Clubs** section is accessible from the Community tab: shows clubs the user belongs to + a "Create Club" CTA + a club search/discover section
10. - [ ] A user can belong to multiple clubs; no hard limit at MVP
11. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DB Schema

```sql
CREATE TABLE clubs (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name          TEXT NOT NULL UNIQUE,
  city          TEXT NOT NULL,
  description   TEXT,
  logo_url      TEXT,
  invite_code   VARCHAR(8) NOT NULL UNIQUE,
  join_policy   TEXT NOT NULL DEFAULT 'approval'
                  CHECK (join_policy IN ('approval', 'open')),
  created_by    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE club_members (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  club_id     UUID NOT NULL REFERENCES clubs(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role        TEXT NOT NULL DEFAULT 'member' CHECK (role IN ('admin', 'member')),
  status      TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'pending', 'removed')),
  joined_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(club_id, user_id)
);

CREATE INDEX idx_clubs_city ON clubs(city);
CREATE INDEX idx_club_members_club_id ON club_members(club_id);
CREATE INDEX idx_club_members_user_id ON club_members(user_id);
```

### Club Posts

Extend the existing `posts` table:
```sql
ALTER TABLE posts ADD COLUMN club_id UUID REFERENCES clubs(id) ON DELETE CASCADE;
```

A post with `club_id IS NOT NULL` is a club-scoped post. The feed query (`GET /api/posts`) already filters; add: if `club_id IS NOT NULL`, only return the post if the requesting user is an active member of that club.

### RLS Consideration

Club posts need an additional RLS policy: `club_id IS NULL OR club_id IN (SELECT club_id FROM club_members WHERE user_id = auth.uid() AND status = 'active')`.

### API Routes

```
GET    /api/clubs                          — list clubs user belongs to + search
POST   /api/clubs                          — create club
GET    /api/clubs/[id]                     — club profile
PATCH  /api/clubs/[id]                     — update settings (admin only)
GET    /api/clubs/[id]/members             — member list
PATCH  /api/clubs/[id]/members/[userId]    — approve/reject/remove/promote (admin)
GET    /api/clubs/join/[code]              — resolve join link
POST   /api/clubs/join/[code]              — request to join
GET    /api/clubs/[id]/posts               — club feed (members only)
```

#### New Finding

**N1 — MUST-FIX — "Join Club" button on profile page broken for non-members**
`ClubProfileView.handleJoin` calls `joinClub(club.inviteCode)`. Since the F2 fix, non-members receive `inviteCode:''` from the server. The call becomes `POST /api/clubs/join/` (empty string) → `resolveJoinLink('')` → NotFoundError → toast "Invalid invite link". Users who navigate to a club profile page via direct URL cannot join. The invite-link flow (`JoinClubView`) is unaffected — it uses the code from the URL param, not the club object.

**Recommendation:** Add `POST /api/clubs/[id]/join` route that resolves the invite code server-side from the club ID, so the client does not need the raw code. `ClubProfileView.handleJoin` can then POST to `/api/clubs/${club.id}/join` without a code.
