# Story 12.2 — Driver's License on Profile

**Epic:** 12 — Profile & Settings
**Story:** 12.2
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 4.6 (DL document management flow), Story 12.1 (profile page)

## Story

**As a** user,
**I want** to see and manage my Driver's License directly from my profile,
**so that** I can quickly check its expiry status and update it when it renews.

---

## Acceptance Criteria

1. - [ ] Profile page has a "Driver's License" section (accessible from action card → DL section, or inline on profile page)
2. - [ ] If no DL exists: "Add Driver's License" CTA → triggers document upload flow with type pre-set to `DL`
3. - [ ] If DL exists: shows expiry date, `StatusBadge` (green=valid/amber=expiring/red=expired), edit + delete options
4. - [ ] Edit DL: reuses document upload flow from Story 4.1/4.6 — navigates to document edit/replace form
5. - [ ] Delete DL: `ConfirmModal` → `DELETE /api/documents/[id]` (same as Story 4.5)
6. - [ ] DL fetched via `GET /api/documents?type=DL` for the authenticated user
7. - [ ] DL included in daily cron expiry check (Story 8.1 already queries all documents including DL — vehicleId may be null for DL)
8. - [ ] `pnpm typecheck` and `pnpm lint` pass

### DL Document Model

[Source: architecture.md#3-database-schema]

Driver's License documents have `vehicle_id = null` in the `documents` table (they are user-level documents, not tied to a specific vehicle). Verify the `documents` table allows `vehicle_id` to be NULL — check the schema.

If the schema has `vehicle_id NOT NULL`, add a migration:
```sql
ALTER TABLE documents ALTER COLUMN vehicle_id DROP NOT NULL;
```

- `status='active'` and expiry <= 30 days: amber "Expiring soon"

### documentService.getUserDocumentByType()

[Source: architecture.md#6-components]

Add to `src/services/documentService.ts` (or use `listByUser` with type param):

```typescript
async getUserDocumentByType(userId: string, type: string): Promise<Document | null> {
  return db.query.documents.findFirst({
    where: and(
      eq(documents.userId, userId),
      eq(documents.type, type),
      isNull(documents.vehicleId),
    ),
    orderBy: [desc(documents.createdAt)],
  }) ?? null;
}
```

Returns the most recently uploaded DL (users may have multiple — return latest).
