### Profile Page Structure

[Source: architecture.md#12-unified-project-structure]

Read existing `src/app/(app)/profile/page.tsx` stub before editing:

```tsx
// src/app/(app)/profile/page.tsx (RSC)
import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";
import { userService } from "@/services/userService";
import { vehicleService } from "@/services/vehicleService";
import { tripService } from "@/services/tripService";
import { communityService } from "@/services/communityService";

export default async function ProfilePage() {
  const session = await getServerSession(authOptions);
  if (!session) redirect("/login");

  const [user, vehicleCount, tripCount, postCount] = await Promise.all([
    userService.getById(session.user.id),
    vehicleService.countByUser(session.user.id),
    tripService.countByUser(session.user.id),
    communityService.countPostsByUser(session.user.id),
  ]);

  return <ProfileView user={user} stats={{ vehicleCount, tripCount, postCount }} />;
}
```

`ProfileView` is a `"use client"` component for the sign-out button interactivity.

### Profile Image Upload

[Source: architecture.md#11-integrations]

Upload flow:
1. User selects image → show local preview
2. On save: `POST /api/upload/profile-image` → returns `{ uploadUrl, key }`
3. `PUT` to R2 pre-signed URL
4. Include `key` in `PATCH /api/users/me { image: key }`

R2 key format: `{userId}/profile/{filename}` (overwrite existing — same key each time for simplicity; OR use timestamp to avoid cache issues).

Create `src/app/api/upload/profile-image/route.ts`:

```typescript
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const { filename, contentType } = await req.json();
    const key = `${session.user.id}/profile/${filename}`;
    const uploadUrl = await generateUploadUrl(key, contentType);
    return NextResponse.json({ uploadUrl, key });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### Username Uniqueness Validation

[Source: architecture.md#5-api-specification]

`GET /api/users/username-check?username=X` (Story 2.2) returns `{ available: boolean }`. In the edit form, debounce 300ms and call this endpoint on username change. Show inline feedback:
- Available: green checkmark
- Taken: red "Username already taken"
- Same as current: skip validation (always valid)
