# Story 8.5 — Notification Preferences in Settings

**Epic:** 08 — Notifications & Background Jobs
**Story:** 8.5
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 8.2 (push subscription infrastructure), Story 8.3 (email notifications field)

---

## Story

**As a** user,
**I want** to control which notifications I receive and how,
**so that** I am only alerted in the ways that work for me.

---

## Acceptance Criteria

1. - [ ] Settings page `src/app/(app)/profile/settings/page.tsx` has a **Notifications** section
2. - [ ] **Expiry window input**: numeric input, default 30, min 7, max 90 days — `PATCH /api/users/me { notificationWindowDays }`
3. - [ ] **Push notifications toggle**: on/off; displays current browser permission status (Granted/Blocked/Not set)
4. - [ ] Push toggle ON: triggers push subscription flow (reuses `usePushSubscription` hook from Story 8.2)
5. - [ ] Push toggle OFF: calls `DELETE /api/push/unsubscribe` which deletes ALL push subscriptions for the user
6. - [ ] **Email notifications toggle**: on/off — `PATCH /api/users/me { emailNotificationsEnabled }`
7. - [ ] All setting changes saved immediately on change (no explicit "Save" button needed for toggles; expiry window saves on blur/Enter)
8. - [ ] Changes to `notificationWindowDays` take effect on the next cron run
9. - [ ] `DELETE /api/push/unsubscribe` — deletes all `push_subscriptions` records for the user
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

### Expiry Window Setting

[Source: front-end-spec.md#5-component-library--ui-patterns]

```tsx
function ExpiryWindowSetting({ value }: { value: number }) {
  const [days, setDays] = useState(value);

  const handleBlur = async () => {
    const clamped = Math.min(90, Math.max(7, days));
    setDays(clamped);
    await fetch("/api/users/me", {
      method: "PATCH",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ notificationWindowDays: clamped }),
    });
  };

  return (
    <div className="flex items-center justify-between px-4 py-4">
      <div>
        <p className="text-sm font-medium text-gray-900">Expiry alert window</p>
        <p className="text-xs text-gray-500">Get warned this many days before expiry</p>
      </div>
      <div className="flex items-center gap-2">
        <input
          type="number"
          min={7}
          max={90}
          value={days}
          onChange={e => setDays(Number(e.target.value))}
          onBlur={handleBlur}
          className="w-16 border border-gray-300 rounded-lg text-center text-sm py-1.5"
        />
        <span className="text-xs text-gray-500">days</span>
      </div>
    </div>
  );
}
```

### Push Notifications Setting

[Source: front-end-spec.md#5-component-library--ui-patterns]

```tsx
function PushNotificationSetting() {
  const { permissionState, subscribe, isSubscribing } = usePushSubscription();
  const [isEnabled, setIsEnabled] = useState(false);

  // Determine enabled state from current subscriptions
  useEffect(() => {
    fetch("/api/push/status").then(r => r.json()).then(d => setIsEnabled(d.hasSubscription));
  }, []);

  const handleToggle = async (on: boolean) => {
    if (on) {
      const success = await subscribe();
      setIsEnabled(success);
    } else {
      await fetch("/api/push/unsubscribe", { method: "DELETE" });
      setIsEnabled(false);
    }
  };

  const permissionLabel = {
    granted: "Permission granted",
    denied: "Blocked in browser — change in browser settings",
    default: "Not yet requested",
    unsupported: "Not supported on this device",
  }[permissionState];

  return (
    <div className="flex items-start justify-between px-4 py-4">
      <div>
        <p className="text-sm font-medium text-gray-900">Push notifications</p>
        <p className="text-xs text-gray-500">{permissionLabel}</p>
      </div>
      <Switch
        checked={isEnabled}
        onCheckedChange={handleToggle}
        disabled={permissionState === "denied" || permissionState === "unsupported" || isSubscribing}
      />
    </div>
  );
}
```

shadcn `Switch` component — check if installed; if not, `pnpm dlx shadcn-ui@latest add switch`.

### Email Toggle

[Source: front-end-spec.md#5-component-library--ui-patterns]

```tsx
function EmailNotificationSetting({
  enabled,
  onToggle,
}: {
  enabled: boolean;
  onToggle: (on: boolean) => Promise<void>;
}) {
  const [isEnabled, setIsEnabled] = useState(enabled);

  const handleToggle = async (on: boolean) => {
    setIsEnabled(on); // Optimistic
    try {
      await onToggle(on);
    } catch {
      setIsEnabled(!on); // Revert on error
    }
  };

  return (
    <div className="flex items-center justify-between px-4 py-4">
      <div>
        <p className="text-sm font-medium text-gray-900">Email notifications</p>
        <p className="text-xs text-gray-500">Document expiry reminders via email</p>
      </div>
      <Switch checked={isEnabled} onCheckedChange={handleToggle} />
    </div>
  );
}
```

`onToggle` calls `PATCH /api/users/me { emailNotificationsEnabled }`.
