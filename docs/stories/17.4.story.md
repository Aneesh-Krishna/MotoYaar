# Story 17.4 — Group Ride: Member View, Panel & Session Lifecycle

**Epic:** 17 — Maps, Group Rides & Offline Navigation
**Story:** 17.4
**Status:** Draft
**Created:** 2026-05-17
**Depends On:** Story 17.3 (Group creation, join, and live broadcast must be working)

## Story

**As a** group ride member,
**I want** to see each rider's distance, direction, ETA, and presence status, and manage the ride session through a group panel,
**so that** I know who is where and whether anyone has fallen behind without having to call anyone.

---

## Acceptance Criteria

1. - [ ] Each group member's marker shows a **direction arrow** (compass bearing from self to that member)
2. - [ ] Each marker shows **distance** from self (straight-line, in km/m) — updated in real time
3. - [ ] Each marker shows a client-side **ETA** (distance ÷ member's reported speed)
4. - [ ] **Presence status indicators** on each marker: green pulse (location < 10s old), yellow (10–60s old), grey + "Last seen X min ago" (offline/backgrounded)
5. - [ ] **Dead reckoning**: if a member's signal is lost, their marker extrapolates position based on last known speed + heading for up to 30 seconds, then freezes
6. - [ ] A **Group Panel** (slide-up drawer) lists all members: avatar, name, speed, last update time, distance from self
7. - [ ] The Group Panel shows a **leader badge** on the group leader
8. - [ ] Group Panel has a "Find on map" button per member — taps centre the map on that member
9. - [ ] Group Panel has "**End ride**" button (leader only) — ends the session for all members
10. - [ ] Group Panel has "**Leave ride**" button (non-leaders) — removes self from session
11. - [ ] **Session ACTIVE state**: leader taps "Start Ride" from WAITING state → all members begin high-frequency location sharing
12. - [ ] **Session ENDED state**: triggered by leader ending ride, OR auto-end after all members offline >5 minutes
13. - [ ] On session end: a **session summary** is generated (total members, duration) and shown to all members
14. - [ ] Invite codes can be disabled by the leader mid-ride to prevent new joins ("**Lock group**" toggle)
15. - [ ] Session lifecycle transitions are reflected in `ride_groups.status` (waiting → active → ended)
16. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Member View — Client-Side Calculations

All distance, bearing, and ETA calculations happen client-side from the Realtime broadcast data — no server round-trip needed.

```typescript
// Haversine distance (metres)
function haversineDistance(lat1, lng1, lat2, lng2): number { ... }

// Compass bearing (degrees, 0 = North)
function bearing(lat1, lng1, lat2, lng2): number {
  const φ1 = toRad(lat1), φ2 = toRad(lat2);
  const Δλ = toRad(lng2 - lng1);
  const y = Math.sin(Δλ) * Math.cos(φ2);
  const x = Math.cos(φ1) * Math.sin(φ2) - Math.sin(φ1) * Math.cos(φ2) * Math.cos(Δλ);
  return (Math.atan2(y, x) * 180 / Math.PI + 360) % 360;
}

// Client-side ETA: distance (km) ÷ speed (km/h) → hours → formatted
function etaMinutes(distanceKm: number, speedKmh: number): number | null {
  if (speedKmh < 1) return null; // stationary — no ETA
  return Math.round((distanceKm / speedKmh) * 60);
}
```

### Dead Reckoning

When a member's broadcast goes stale (no update received for >3s), extrapolate position:

```typescript
function extrapolatePosition(
  lastLat: number, lastLng: number,
  speedKmh: number, headingDeg: number,
  elapsedMs: number
): { lat: number; lng: number } {
  const distanceKm = (speedKmh / 3600) * (elapsedMs / 1000);
  const R = 6371; // Earth radius km
  const δ = distanceKm / R;
  const θ = (headingDeg * Math.PI) / 180;
  const φ1 = (lastLat * Math.PI) / 180;
  const λ1 = (lastLng * Math.PI) / 180;
  const φ2 = Math.asin(Math.sin(φ1)*Math.cos(δ) + Math.cos(φ1)*Math.sin(δ)*Math.cos(θ));
  const λ2 = λ1 + Math.atan2(Math.sin(θ)*Math.sin(δ)*Math.cos(φ1), Math.cos(δ)-Math.sin(φ1)*Math.sin(φ2));
  return { lat: φ2 * 180 / Math.PI, lng: λ2 * 180 / Math.PI };
}
// Stop extrapolating after 30 seconds — freeze marker at last extrapolated position
```

### Presence Status Logic

```typescript
function getPresenceStatus(lastSeenTs: number): 'fresh' | 'stale' | 'offline' {
  const ageMs = Date.now() - lastSeenTs;
  if (ageMs < 10_000) return 'fresh';   // green pulse
  if (ageMs < 60_000) return 'stale';   // yellow
  return 'offline';                     // grey + last seen X min
}
```

### Session Lifecycle API

```
PATCH /api/ride-groups/[id]/start   — leader only; status: waiting → active
PATCH /api/ride-groups/[id]/end     — leader only; status: active → ended; set ended_at
```

Auto-end: a Supabase Edge Function or Vercel cron checks for groups where all `ride_sessions.last_seen_at` is >5 minutes ago and status is `active` → auto-end. For MVP, implement as a periodic API check from the leader's browser (every 60s, check if all members offline >5min, then call `/end`). Server-side auto-end is Phase 2.
