# Story 17.6 — Offline: MBTiles Packaging, OPFS Storage & R2 Delivery

**Epic:** 17 — Maps, Group Rides & Offline Navigation
**Story:** 17.6
**Status:** Draft
**Created:** 2026-05-17

## Story

**As a** rider who has confirmed an offline route download,
**I want** the app to store the map tiles, route graph, and voice clips in my browser's local storage,
**so that** I can navigate offline with no internet connection.

---

## Acceptance Criteria

1. - [ ] Tiles collected by the Edge Function are **packaged into MBTiles format** (SQLite-backed, gzipped PBF) server-side
2. - [ ] The MBTiles package is uploaded to **Cloudflare R2** and a **pre-signed URL** (1-hour expiry) is returned to the client
3. - [ ] The route graph (MessagePack + gzip) and voice clips (MP3 array) are bundled into the same R2 package or as separate files alongside MBTiles
4. - [ ] The client downloads the package via the pre-signed URL and stores tiles in **OPFS** (`navigator.storage.getDirectory()`) — not IndexedDB
5. - [ ] Before download begins, the app **checks available browser storage quota**; if insufficient, it offers a "Basic Download" option (zoom 8–13 only, ~8–12 MB) with a clear explanation
6. - [ ] On iOS < 15.2, the "Download for Offline" button is **not shown**; a clear message is displayed: "Offline navigation requires iOS 15.2 or later. Update your device to use this feature."
7. - [ ] Download progress is shown in the persistent bottom bar from Story 17.5
8. - [ ] On completion, `offline_routes.opfs_key` is updated with the OPFS directory key for this route
9. - [ ] Maximum **5 stored offline routes** is enforced — when the user tries to add a 6th, an LRU eviction warning is shown listing which route will be removed, with confirmation required
10. - [ ] The **Offline Routes management screen** lists all stored routes: name, download date, expiry date, size, and a delete button
11. - [ ] Deleting a route removes the OPFS data and the `offline_routes` record
12. - [ ] Routes approaching expiry (within 3 days) show a warning badge on the management screen
13. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### MBTiles Packaging (Edge Function — continuation of Story 17.5)

MBTiles is a SQLite database containing map tiles. Server-side in Deno, use `sql.js` (WASM SQLite) or write raw binary to produce a valid MBTiles file:

```typescript
// MBTiles schema (standard)
// CREATE TABLE tiles (zoom_level INTEGER, tile_column INTEGER, tile_row INTEGER, tile_data BLOB);
// CREATE TABLE metadata (name TEXT, value TEXT);
// CREATE UNIQUE INDEX tile_index ON tiles (zoom_level, tile_column, tile_row);
// Note: tile_row is TMS y-flip: tile_row = (2^zoom - 1) - y
```

For Deno: use `npm:better-sqlite3` or write raw bytes. Alternative: produce a zip archive of tile files (simpler but larger). **MBTiles is preferred** — it's the standard and works with MapLibre GL JS natively.

[Source: PRD Section 7.3.2]

### OPFS Storage Strategy

OPFS (`navigator.storage.getDirectory()`) provides a sandboxed file system accessible only to this origin. Tiles are stored here, not IndexedDB (IndexedDB is inefficient for binary blob storage at this scale).

```typescript
// src/lib/offline/opfsStorage.ts
export async function saveToOPFS(key: string, data: ArrayBuffer): Promise<void> {
  const root = await navigator.storage.getDirectory();
  const dirHandle = await root.getDirectoryHandle('offline-routes', { create: true });
  const fileHandle = await dirHandle.getFileHandle(key, { create: true });
  const writable = await fileHandle.createWritable();
  await writable.write(data);
  await writable.close();
}

export async function readFromOPFS(key: string): Promise<ArrayBuffer | null> {
  const root = await navigator.storage.getDirectory();
  const dirHandle = await root.getDirectoryHandle('offline-routes', { create: false });
  const fileHandle = await dirHandle.getFileHandle(key);
  const file = await fileHandle.getFile();
  return file.arrayBuffer();
}

export async function deleteFromOPFS(key: string): Promise<void> {
  const root = await navigator.storage.getDirectory();
  const dirHandle = await root.getDirectoryHandle('offline-routes', { create: false });
  await dirHandle.removeEntry(key);
}
```

### iOS OPFS Feature Detection (Non-Negotiable)

```typescript
// Before showing "Download for Offline" button:
export async function isOPFSSupported(): Promise<boolean> {
  try {
    await navigator.storage.getDirectory();
    return true;
  } catch {
    return false;
  }
}
```

If `isOPFSSupported()` returns false, hide the button and show: "Offline navigation requires iOS 15.2 or later. Update your device to use this feature."

Do NOT show the download button on unsupported devices. No silent failures.

[Source: PRD Section 8 — iOS Limitations]

### Storage Quota Check

```typescript
export async function checkStorageQuota(requiredBytes: number): Promise<{
  sufficient: boolean;
  availableBytes: number;
}> {
  const estimate = await navigator.storage.estimate();
  const available = (estimate.quota ?? 0) - (estimate.usage ?? 0);
  return { sufficient: available >= requiredBytes, availableBytes: available };
}
```

If insufficient: offer "Basic Download" (zoom 8–13 only, approximately 8–12 MB). Standard is zoom 8–15 (~17–35 MB for 100km route).

[Source: PRD Section 7.3.3, PRD Section 7.3.8]

### R2 Upload (from Edge Function)

```typescript
// In Edge Function: upload assembled package to R2
import { S3Client, PutObjectCommand } from 'npm:@aws-sdk/client-s3';
import { getSignedUrl } from 'npm:@aws-sdk/s3-request-presigner';

const s3 = new S3Client({ /* R2 config */ });
const key = `offline/${routeId}/package.mbtiles`;
await s3.send(new PutObjectCommand({ Bucket: R2_BUCKET, Key: key, Body: mbtiles }));
const downloadUrl = await getSignedUrl(s3, new GetObjectCommand({ Bucket: R2_BUCKET, Key: key }), { expiresIn: 3600 });
```

The same R2 client pattern is already used for document storage in the existing architecture.
