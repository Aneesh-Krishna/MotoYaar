# Story 9.5 — Report Post

**Epic:** 09 — Community
**Story:** 9.5
**Status:** Draft
**Created:** 2026-03-15

## Story

**As a** logged-in user,
**I want** to report a post I believe violates community guidelines,
**so that** the moderation team can review and take appropriate action.

---

## Acceptance Criteria

1. - [ ] "Report" option in post detail kebab menu — visible to logged-in users who do NOT own the post
2. - [ ] Report modal/form: reason dropdown (Spam / Inappropriate / Misinformation / Harassment / Other) + optional description textarea
3. - [ ] `POST /api/posts/[id]/reports { reason, description? }` — creates report record
4. - [ ] `communityService.reportPost(postId, reporterUserId, reason, description)` in `src/services/communityService.ts`
5. - [ ] `post_reports` table has `UNIQUE(post_id, reporter_user_id)` — duplicate report from same user returns 409 but is shown silently (no error toast shown to user; just dismiss modal)
6. - [ ] Auto-hide: after submitting report, COUNT unique reports for the post; if `>= admin_settings.auto_hide_report_threshold` (default 10): set `post.is_hidden=true`
7. - [ ] Auto-hidden post shows "This post has been hidden pending review." when navigated to directly
8. - [ ] Admin threshold read from `admin_settings` table (key `auto_hide_report_threshold`, value `10`)
9. - [ ] On success: toast "Thanks for reporting. We'll review this post."
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

CREATE TABLE post_reports (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  post_id         UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  reporter_user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,

  key    TEXT PRIMARY KEY,
  value  TEXT NOT NULL
);

  ON CONFLICT DO NOTHING;

### communityService.reportPost()

[Source: architecture.md#6-components]

Add to `src/services/communityService.ts`:

```typescript
async reportPost(
  postId: string,
  reporterUserId: string,
  reason: string,
  description?: string
): Promise<void> {
  // Insert report (UNIQUE constraint handles duplicates)
  try {
    await db.insert(postReports).values({
      postId,
      reporterUserId,
      reason,
      description: description || null,
    });
  } catch (err) {
    if (isUniqueConstraintError(err)) {
      // Duplicate report — silently ignore (do not throw)
      return;
    }
    throw err;
  }

  // Auto-hide check
  const [{ count }] = await db
    .select({ count: sql<number>`COUNT(DISTINCT reporter_user_id)` })
    .from(postReports)
    .where(eq(postReports.postId, postId));

  const threshold = await this.getAdminSetting("auto_hide_report_threshold", 10);

  if (Number(count) >= threshold) {
    await db.update(posts)
      .set({ isHidden: true })
      .where(eq(posts.id, postId));
  }
}

private async getAdminSetting(key: string, defaultValue: number): Promise<number> {
  const setting = await db.query.adminSettings.findFirst({
    where: eq(adminSettings.key, key),
  });
  return setting ? parseInt(setting.value, 10) : defaultValue;
}
```

`isUniqueConstraintError(err)` checks `err.code === '23505'` (PostgreSQL unique violation).

### Report Modal Component

[Source: front-end-spec.md#5-component-library--ui-patterns]

Create as a Dialog/modal using shadcn `Dialog`:

```tsx
// In PostDetailView (or as separate component)
function ReportModal({
  postId,
  open,
  onClose,
}: {
  postId: string;
  open: boolean;
  onClose: () => void;
}) {
  const [reason, setReason] = useState("");
  const [description, setDescription] = useState("");
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setIsSubmitting(true);
    const res = await fetch(`/api/posts/${postId}/reports`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ reason, description }),
    });
    setIsSubmitting(false);
    onClose();
    // Show success toast regardless of 409 (duplicate handled silently)
    if (res.ok || res.status === 409) {
      toast.success("Thanks for reporting. We'll review this post.");
    }
  };

  return (
    <Dialog open={open} onOpenChange={onClose}>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>Report post</DialogTitle>
        </DialogHeader>
        <form onSubmit={handleSubmit} className="space-y-4">
          <Select value={reason} onValueChange={setReason} required>
            <SelectTrigger><SelectValue placeholder="Select a reason" /></SelectTrigger>
            <SelectContent>
              <SelectItem value="spam">Spam</SelectItem>
              <SelectItem value="inappropriate">Inappropriate</SelectItem>
              <SelectItem value="misinformation">Misinformation</SelectItem>
              <SelectItem value="harassment">Harassment</SelectItem>
              <SelectItem value="other">Other</SelectItem>
            </SelectContent>
          </Select>
          <Textarea
            placeholder="Additional details (optional)"
            value={description}
            onChange={e => setDescription(e.target.value)}
            rows={3}
          />
          <Button type="submit" disabled={!reason || isSubmitting} className="w-full">
            {isSubmitting ? "Submitting..." : "Submit report"}
          </Button>
        </form>
      </DialogContent>
    </Dialog>
  );
}
```

### POST /api/posts/[id]/reports

[Source: architecture.md#5-api-specification]

Create `src/app/api/posts/[id]/reports/route.ts`:

```typescript
import { z } from "zod";

const reportSchema = z.object({
  reason: z.enum(["spam", "inappropriate", "misinformation", "harassment", "other"]),
  description: z.string().max(500).optional(),
});

export async function POST(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const body = await req.json();
    const { reason, description } = reportSchema.parse(body);

    await communityService.reportPost(params.id, session.user.id, reason, description);
    return NextResponse.json({ ok: true });
  } catch (error) {
    return handleApiError(error);
  }
}
```

`ConflictError` would normally map to 409. However, since duplicate reports are silently swallowed in the service layer, the route always returns 200. Ensure service layer handles the unique constraint before it surfaces as an unhandled error.
