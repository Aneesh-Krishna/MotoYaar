# Story 9.4 — Edit Post

**Epic:** 09 — Community
**Story:** 9.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 9.3 (post detail with kebab menu)

---

## Story

**As a** post author,
**I want** to edit my post after publishing it,
**so that** I can fix mistakes or update information.

---

## Acceptance Criteria

1. - [ ] Edit option accessible from post detail kebab menu (own posts only)
2. - [ ] Edit form pre-populated with all current post values (title, description, images, link, tags)
3. - [ ] Same `PostForm` component used for editing (from Story 9.1)
4. - [ ] On save: `PATCH /api/posts/[id]` updates the post
5. - [ ] `post.edited` set to `true` on first edit
6. - [ ] `post.edited_at` updated to current timestamp
7. - [ ] Previous title and description saved to `post.edit_history` JSONB array (appended, not replaced)
8. - [ ] "Edited" label appears immediately on post detail after save (post detail re-fetches or local state updated)
9. - [ ] If user does not own the post: `PATCH /api/posts/[id]` returns 403 (`ForbiddenError`)
10. - [ ] No time limit on editing (user can edit any time)
11. - [ ] `communityService.updatePost(postId, userId, data)` — performs ownership check
12. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Edit Route

[Source: architecture.md#12-unified-project-structure]

Two options for the edit UI:
- Navigate to `src/app/(app)/community/[id]/edit/page.tsx` (recommended for MVP — easier back navigation)
- Open an in-place modal/sheet

Use the page approach. Create `src/app/(app)/community/[id]/edit/page.tsx`:

```tsx
// src/app/(app)/community/[id]/edit/page.tsx (RSC)
export default async function EditPostPage({ params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) redirect("/login");

  const post = await communityService.getPost(params.id, session.user.id);

  // Ownership guard at page level
  if (post.userId !== session.user.id) redirect(`/community/${params.id}`);

  return <EditPostView post={post} />;
}
```

`EditPostView` is a `"use client"` component that renders `PostForm` with `initialValues` from the existing post and a handler that calls `PATCH /api/posts/[id]`.

export async function PATCH(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const body = await req.json();
    const data = createPostSchema.parse(body);
    const post = await communityService.updatePost(params.id, session.user.id, data);
    return NextResponse.json(post);
  } catch (error) {
    return handleApiError(error);
  }
}

### communityService.updatePost()

[Source: architecture.md#6-components]

Add to `src/services/communityService.ts`:

```typescript
async updatePost(postId: string, userId: string, data: CreatePostInput): Promise<Post> {
  const existing = await db.query.posts.findFirst({
    where: eq(posts.id, postId),
  });
  if (!existing) throw new NotFoundError("Post not found");
  if (existing.userId !== userId) throw new ForbiddenError("You do not own this post");

  // Append current values to edit_history before overwriting
  const editEntry = {
    title: existing.title,
    description: existing.description,
    editedAt: new Date().toISOString(),
  };
  const editHistory = Array.isArray(existing.editHistory)
    ? [...existing.editHistory, editEntry]
    : [editEntry];

  const [updated] = await db.update(posts)
    .set({
      title: data.title,
      description: data.description,
      imageUrls: data.imageKeys ?? existing.imageUrls,
      link: data.link || null,
      tags: data.tags ?? existing.tags,
      edited: true,
      editedAt: new Date(),
      editHistory,
    })
    .where(eq(posts.id, postId))
    .returning();

  return updated;
}
```

### edit_history JSONB Structure

[Source: architecture.md#3-database-schema]

Each entry in `edit_history` (stored as JSONB array in `posts.edit_history`):

```typescript
interface EditHistoryEntry {
  title: string;
  description: string;
  editedAt: string; // ISO 8601
}
```

The array is append-only. The current (latest) values are always in the main columns. History allows reviewing what was changed. For MVP, this data is stored but not displayed to users.

### "Edited" Label Update

[Source: front-end-spec.md#5-component-library--ui-patterns]

After a successful PATCH, the client navigates back to the post detail page (`/community/[id]`). The page re-fetches the post (it is an RSC), so `post.edited=true` will cause the "Edited" label to render automatically. No special local state needed.
