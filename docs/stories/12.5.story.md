# Story 12.5 — Privacy, Data & Account Deletion

**Epic:** 12 — Profile & Settings
**Story:** 12.5
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 12.4 (settings page with Privacy & Data section)

## Story

**As a** user who wants to leave MotoYaar or manage my stored data,
**I want** to control what is stored and permanently delete my account if I choose,
**so that** I have full control over my personal information.

---

## Acceptance Criteria

1. - [ ] Settings → Privacy & Data section has:
   - Document storage preference toggle (parse-only / store)
   - Privacy Policy link (→ `/privacy` static page)
   - "Delete all stored documents" button
   - "Delete my account" button
2. - [ ] **Document storage preference toggle**: `PATCH /api/users/me { documentStoragePreference: "store" | "parse_only" }` — when `parse_only`, new documents are parsed but file not stored in R2
3. - [ ] **"Delete all stored documents"**: `ConfirmModal` → `DELETE /api/documents/stored` → deletes all R2 files for the user's documents where `storage_url IS NOT NULL` + sets `storage_url=null` on those records
4. - [ ] **"Delete my account"**: `ConfirmModal` requiring user to type "DELETE" (case-sensitive) to enable confirm button
5. - [ ] On confirm: `POST /api/users/me/delete` (or `DELETE /api/users/me`)
6. - [ ] Server-side account deletion: (a) delete all R2 files (profile image, documents, receipts, post images), (b) cascade DB delete, (c) sign user out
7. - [ ] After deletion: redirect to `/login` with query param `?message=deleted`; login page shows "Your account has been deleted."
8. - [ ] `/privacy` static page: minimal content explaining storage, access, encryption, auto-delete policy
9. - [ ] `userService.deleteAccount(userId)` in `src/services/userService.ts`
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

```tsx
function DocumentStorageSetting({ preference }: { preference: string }) {
  const [pref, setPref] = useState(preference);

  const handleToggle = async (storeEnabled: boolean) => {
    const value = storeEnabled ? "store" : "parse_only";
    setPref(value);
    await fetch("/api/users/me", {
      method: "PATCH",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ documentStoragePreference: value }),
    });

  return (
    <div className="px-4 py-4">
      <div className="flex items-center justify-between">
        <div>
          <p className="text-sm font-medium text-gray-900">Store document files</p>
          <p className="text-xs text-gray-500 mt-0.5">
            Keep original files in secure storage for future access
          </p>
        </div>
        <Switch checked={pref === "store"} onCheckedChange={handleToggle} />
      </div>
      {pref === "parse_only" && (
        <p className="text-xs text-amber-600 mt-2">
          New documents will be parsed but files won't be stored.
          Existing stored files are not affected.
        </p>
      )}
    </div>
  );
}
```

`documentService.deleteAllStoredFiles(userId)`:

```typescript
async deleteAllStoredFiles(userId: string): Promise<number> {
  const docsWithFiles = await db.query.documents.findMany({
    where: and(eq(documents.userId, userId), isNotNull(documents.storageUrl)),
  });

  let deleted = 0;
  for (const doc of docsWithFiles) {
    try {
      await deleteObject(doc.storageUrl!);
      deleted++;
    } catch (err) {
      logger.warn({ err, docId: doc.id }, "R2 delete failed");
    }
    await db.update(documents)
      .set({ storageUrl: null })
      .where(eq(documents.id, doc.id));
  }
  return deleted;
}
```

### Account Deletion ConfirmModal

[Source: front-end-spec.md#5-component-library--ui-patterns]

The delete account confirmation requires typing "DELETE":

```tsx
function DeleteAccountAction() {
  const [isOpen, setIsOpen] = useState(false);
  const [confirmText, setConfirmText] = useState("");
  const [isDeleting, setIsDeleting] = useState(false);
  const router = useRouter();

  const handleDelete = async () => {
    if (confirmText !== "DELETE") return;
    setIsDeleting(true);
    const res = await fetch("/api/users/me/delete", { method: "POST" });
    if (res.ok) {
      await signOut({ redirect: false });
      router.push("/login?message=deleted");
    }
  };

  return (
    <>
      <button
        onClick={() => setIsOpen(true)}
        className="w-full flex items-center justify-between px-4 py-4 text-left"
      >
        <div>
          <p className="text-sm font-medium text-red-600">Delete my account</p>
          <p className="text-xs text-gray-500">Permanently delete all data</p>
        </div>
        <ChevronRight size={16} className="text-gray-400" />
      </button>

      <Dialog open={isOpen} onOpenChange={setIsOpen}>
        <DialogContent className="max-w-sm">
          <DialogHeader>
            <DialogTitle className="text-red-600">Delete account</DialogTitle>
          </DialogHeader>
          <div className="space-y-4">
            <p className="text-sm text-gray-600">
              This will permanently delete your account, all vehicles, documents, expenses, trips,
              and community posts. This action cannot be undone.
            </p>
            <p className="text-sm font-medium text-gray-700">Type DELETE to confirm:</p>
            <Input
              value={confirmText}
              onChange={e => setConfirmText(e.target.value)}
              placeholder="DELETE"
              className="font-mono"
            />
            <Button
              variant="destructive"
              className="w-full"
              disabled={confirmText !== "DELETE" || isDeleting}
              onClick={handleDelete}
            >
              {isDeleting ? "Deleting..." : "Delete my account"}
            </Button>
          </div>
        </DialogContent>
      </Dialog>
    </>
  );
}
```

### userService.deleteAccount()

[Source: architecture.md#6-components]

Add to `src/services/userService.ts`:

```typescript
async deleteAccount(userId: string): Promise<void> {
  logger.info({ userId }, "Account deletion initiated");

  // 1. Collect all R2 keys to delete
  const [user, docs, userPosts] = await Promise.all([
    db.query.users.findFirst({ where: eq(users.id, userId) }),
    db.query.documents.findMany({ where: eq(documents.userId, userId) }),
    db.query.posts.findMany({ where: eq(posts.userId, userId) }),
  ]);

  // 2. Delete R2 files (non-fatal per-file)
  const keysToDelete: string[] = [];
  if (user?.image) keysToDelete.push(user.image); // profile image
  docs.forEach(d => { if (d.storageUrl) keysToDelete.push(d.storageUrl); });
  userPosts.forEach(p => { p.imageUrls?.forEach(k => keysToDelete.push(k)); });

  await Promise.allSettled(
    keysToDelete.map(key =>
      deleteObject(key).catch(err => logger.warn({ err, key }, "R2 delete failed in account deletion"))
    )
  );

  // 3. DB cascade delete — ON DELETE CASCADE handles:
  //    vehicles → documents, expenses, trips, vehicle_access, vehicle_invites
  //    posts → reactions, comments, reports
  //    notifications, push_subscriptions, ai_reports
  await db.delete(users).where(eq(users.id, userId));

  logger.info({ userId }, "Account deleted");
}
```

The `ON DELETE CASCADE` on all foreign keys pointing to `users.id` handles the DB cleanup. Verify cascade constraints in the schema before relying on this.
