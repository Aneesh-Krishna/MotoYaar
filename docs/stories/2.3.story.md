# Story 2.3 — Onboarding Walkthrough Modal

**Epic:** 02 — Authentication & Onboarding
**Story:** 2.3
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 2.2 (Onboarding form)

---

## Story

**As a** new user who just completed profile setup,
**I want** a brief tour of the app's key sections,
**so that** I know where everything is before I start exploring.

---

## Acceptance Criteria

1. - [ ] 5-card modal shown after onboarding form completion (navigated to automatically)
2. - [ ] Cards cover: Dashboard, Garage, Community, Trips, Profile — one section per card
3. - [ ] Each card shows: section name (heading) + one-line description + icon/illustration placeholder
4. - [ ] Copy: *"Your garage, organized."* / *"Every rupee, accounted for."* / *"Log every journey."* / *"Connect with your tribe."* / *"All of it, in your pocket."*
5. - [ ] Navigation: "Next" / "Back" (on cards 2–5) / "Skip" (closes modal immediately)
6. - [ ] Final card (card 5) has "Get started" button instead of "Next"
7. - [ ] On completion or skip: `walkthrough_seen = true` saved to user record; user redirected to Dashboard
8. - [ ] Walkthrough never auto-shows again after completion or skip

## Dev Notes

### Walkthrough Route

[Source: front-end-spec.md#4.1-onboarding--auth]

The walkthrough can be implemented as:
- A dedicated page at `/onboarding/walkthrough` (recommended — shareable URL, re-accessible from Settings)

**Recommended approach:** Dedicated page at `src/app/(auth)/walkthrough/page.tsx`. After completion/skip → `redirect("/")` (Dashboard).

### Walkthrough UI Spec

[Source: front-end-spec.md#4.1-onboarding--auth]

- Full-screen overlay (not a bottom sheet)
- Card-style layout: illustration/icon (top 50%) + section name (H2) + one-line copy (Body) (bottom 50%)
- Progress dots at bottom: 5 dots, filled dot = current card
- Navigation buttons:
  - Card 1: `Skip` (right-aligned) + `Next` (primary CTA)
  - Cards 2–4: `Back` (ghost/secondary) + `Skip` + `Next`
  - Card 5: `Back` + `Get started` (primary CTA; no Skip)
- Animations: slide transition between cards (Framer Motion — already installed)
- Background: white card on dim overlay (or full-screen white for simplicity)

---

### Saving Walkthrough Seen State

[Source: architecture.md#4-data-models]

When user clicks "Get started" or "Skip", call `PATCH /api/users/me` with `{ walkthroughSeen: true }`.

The `walkthroughSeen` field already exists in the `users` table and `session.user` (added in Story 2.1).

After saving, use `router.push("/")` to navigate to Dashboard.

---

### DB Schema Reference

[Source: architecture.md#9-database-schema]

```sql
-- users table (already has this column)
walkthrough_seen BOOLEAN NOT NULL DEFAULT false
```

Update `updateUserSchema` in `src/lib/validations/user.ts` to include `walkthroughSeen`:
```typescript
walkthroughSeen: z.boolean().optional(),
```

### Re-access from Settings

[Source: prd.md#8.11-user-profile--settings]

The walkthrough must be re-accessible from Settings. This is handled in Story 2.4. For this story, just ensure the walkthrough page is a standalone route (`/onboarding/walkthrough` or `/walkthrough`) that can be navigated to directly — not embedded in a flow that requires coming from the onboarding form.
