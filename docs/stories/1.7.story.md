## Dev Notes

### Vercel Deployment Overview

[Source: architecture.md#14-deployment-architecture]

| Environment | URL | Purpose |
|-------------|-----|---------|
| Development | `http://localhost:3000` | Local development |
| Preview | `https://motoyaar-[branch]-[org].vercel.app` | Per-PR auto-deploy |
| Production | `https://motoyaar.app` | Live (main branch) |

- **Build Command:** `next build` (auto-detected by Vercel)
- **Node.js Runtime:** 20 (set in Vercel project settings)
- **Serverless Functions:** API routes deployed as Vercel Serverless Functions

---

### Vercel Setup Steps

[Source: architecture.md#14-deployment-architecture]

1. Create a Vercel account (free) at vercel.com
2. Import the Git repository → Vercel auto-detects Next.js
3. Set Node.js version to **20** in Project Settings → General
4. Add all environment variables (see below)
5. Deploy — first deployment to `main` creates the production URL

---

### Environment Variables to Add in Vercel

[Source: architecture.md#required-environment-variables]

Add ALL of the following to Vercel Project Settings → Environment Variables (Production + Preview):

```
NEXTAUTH_URL          = https://motoyaar.app (production) / https://[preview-url] (preview)
NEXTAUTH_SECRET       = [generate: openssl rand -base64 32]
GOOGLE_CLIENT_ID      = [from Google Cloud Console]
GOOGLE_CLIENT_SECRET  = [from Google Cloud Console]

DATABASE_URL          = [pgBouncer pooler URL from Supabase — port 6543]
                        postgresql://postgres.[project]:[password]@aws-0-ap-south-1.pooler.supabase.com:6543/postgres

R2_ACCOUNT_ID         = [Cloudflare Account ID]
R2_ACCESS_KEY_ID      = [R2 API token access key]
R2_SECRET_ACCESS_KEY  = [R2 API token secret]
R2_BUCKET_NAME        = motoyaar-documents
R2_PUBLIC_URL         = [optional: public R2 URL if configured]

RESEND_API_KEY        = [Resend API key]

ANTHROPIC_API_KEY     = [Anthropic API key]

VAPID_PUBLIC_KEY      = [generated VAPID public key]
VAPID_PRIVATE_KEY     = [generated VAPID private key]
VAPID_EMAIL           = mailto:support@motoyaar.app
NEXT_PUBLIC_VAPID_PUBLIC_KEY = [same as VAPID_PUBLIC_KEY]

CRON_SECRET           = [generate: openssl rand -base64 32]

ADMIN_EMAIL           = admin@motoyaar.app
ADMIN_PASSWORD        = [strong password]
```

> ⚠️ For production `DATABASE_URL`: use the **pgBouncer pooler URL** (port 6543), not the direct connection URL (port 5432). Serverless functions require connection pooling to avoid exhausting Postgres connections.

### `vercel.json` — Cron Configuration

[Source: architecture.md#vercel-json-cron-configuration]

Verify `vercel.json` exists in project root (created in Story 1.1):

```json
{
  "crons": [
    {
      "path": "/api/cron/document-expiry",
      "schedule": "30 0 * * *"
    }
  ]
}
```

> `30 0 * * *` = 00:30 UTC = ~06:00 IST.

### Cron Route Stub

[Source: architecture.md#12-unified-project-structure]

The cron endpoint `src/app/api/cron/document-expiry/route.ts` must exist for Vercel to register the cron job. If not yet created, create a stub:

```typescript
import { NextRequest, NextResponse } from "next/server";

// TODO: Implement full cron logic in Story 8.1
export async function GET(req: NextRequest) {
  const authHeader = req.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  return NextResponse.json({
    message: "Cron stub — not yet implemented",
    processed: 0,
    notified: 0,
    deleted: 0,
  });
}
```
