# Story 7.6 — Currency Conversion for Reports

**Epic:** 07 — Reports & Analytics
**Story:** 7.6
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 7.2 (overall report uses currency), Story 5.1 (expenses store currency field)

---

## Story

**As a** user who has recorded expenses in multiple currencies or changed their currency setting,
**I want** all report amounts converted into my current preferred currency,
**so that** I see consistent totals without manually performing conversions.

---

## Acceptance Criteria

1. - [ ] All expense amounts stored with their original currency (`currency TEXT`) and original amount (`price NUMERIC`) — this is already in the schema from Story 5.1; confirm before assuming
2. - [ ] On report generation: `convertAmount(amount, fromCurrency, toCurrency)` from `src/utils/currency.ts` (Story 7.2) is applied to convert all amounts to the user's current `currency` setting
3. - [ ] A conversion note is shown in the report header if any amount was converted (i.e. `fromCurrency !== toCurrency` for at least one expense)
4. - [ ] If the user has always used INR (default) and never changed currency: no conversion note is shown
5. - [ ] Currency label (e.g. "₹", "USD") is displayed next to all price inputs across the app
6. - [ ] Changing currency via `PATCH /api/users/me { currency }` (already built in Story 2.2) takes effect immediately in reports on next load
7. - [ ] Currency setting persisted in `users.currency` column (default `'INR'`)
8. - [ ] `src/utils/currency.ts` supports: INR, USD, EUR, GBP, AED, SGD — static exchange rates (MVP)
9. - [ ] `pnpm typecheck` and `pnpm lint` pass

### currency.ts (from Story 7.2)

[Source: architecture.md#section-currency]

`src/utils/currency.ts` is created in Story 7.2. This story extends it with additional helpers:

```typescript
// Already in currency.ts from Story 7.2:
// convertAmount(amount, fromCurrency, toCurrency): number
// formatCurrency(amount, currency): string

// Add in this story:

/** Returns the currency symbol for display in price inputs */
export function getCurrencySymbol(currency: string): string {
  const symbols: Record<string, string> = {
    INR: "₹",
    USD: "$",
    EUR: "€",
    GBP: "£",
    AED: "د.إ",
    SGD: "S$",
  };
  return symbols[currency] ?? currency;
}

/** Returns true if any expense in the list has a currency different from targetCurrency */
export function hadCurrencyConversion(
  expenses: Array<{ currency: string }>,
  targetCurrency: string
): boolean {
  return expenses.some(e => e.currency !== targetCurrency);
}
```

### Applying Conversion in reportService

[Source: architecture.md#6-components]

In `src/services/reportService.ts`, when building report totals:

```typescript
// Get user's currency setting
const user = await userService.getById(userId);
const targetCurrency = user.currency ?? "INR";

// Convert each expense amount
const convertedExpenses = expenses.map(e => ({
  ...e,
  convertedAmount: convertAmount(e.price, e.currency, targetCurrency),
}));

const totalSpend = convertedExpenses.reduce((s, e) => s + e.convertedAmount, 0);
const hadConversion = hadCurrencyConversion(expenses, targetCurrency);
```

The `convertedAmount` is used for all aggregations. The original `price` is preserved for display if needed.

[Source: architecture.md#5-api-specification]

- The session must include the updated currency. If `session.user.currency` is used in client components, the session should be refreshed via `update()` from `next-auth/react`.
- Report pages re-fetch on load, so they automatically use the new setting.
