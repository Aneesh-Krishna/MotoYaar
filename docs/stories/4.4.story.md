# Story 4.4 — Document Storage Preference

**Epic:** 04 — Document Management
**Story:** 4.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 4.1 (storage preference affects upload behavior)

---

## Story

**As a** privacy-conscious user,
**I want** control over whether my documents are stored after parsing,
**so that** I can trust MotoYaar with sensitive legal documents.

---

## Acceptance Criteria

1. - [ ] Default storage behavior is **parse-only**: document file is deleted from R2 immediately after AI parsing; only the expiry date is persisted in the DB
2. - [ ] Opt-in: user can enable **full storage** — document file is kept in R2 after parsing, accessible via signed URL (15-min TTL)
3. - [ ] Storage preference toggle is in the Settings page: `src/app/(app)/profile/settings/page.tsx` — add a "Document Storage" section with a toggle and explanatory text
4. - [ ] `PATCH /api/users/me` (or `/api/users/me/settings`) with `{ documentStoragePreference: "parse_only" | "full_storage" }` updates the user's preference
5. - [ ] Stored documents are shown with a "Stored" badge in `DocumentRow`; non-stored shown with "Parsed only" badge (small gray pill)
6. - [ ] Signed URL generation: `generateAccessUrl(storageKey, 900)` (15-min TTL = 900 seconds) from `src/lib/r2.ts`; used by `GET /api/documents/[id]/url` route
7. - [ ] Privacy reminder shown during document upload confirmation screen (Screen 3 from Story 4.1): text indicating current preference + link to settings
8. - [ ] Auto-delete 10 days post-expiry: document `storageKey` is used by the cron job (Story 8.1) — this story ensures `storageKey` is stored correctly in the DB column; the cron itself is not implemented here
9. - [ ] `pnpm typecheck` and `pnpm lint` pass

### DocumentStorageSection Component

[Source: front-end-spec.md#4-core-screen-specifications]

```tsx
"use client";
import { Switch } from "@/components/ui/switch";

interface Props {
  currentPreference: "parse_only" | "full_storage";
}

export function DocumentStorageSection({ currentPreference }: Props) {
  const [pref, setPref] = useState(currentPreference);
  const [isSaving, setIsSaving] = useState(false);

  const handleToggle = async (checked: boolean) => {
    const newPref = checked ? "full_storage" : "parse_only";
    setIsSaving(true);
    try {
      await userApi.updateSettings({ documentStoragePreference: newPref });
      setPref(newPref);
      toast.success("Storage preference updated");
    } catch {
      toast.error("Failed to update preference");
    } finally {
      setIsSaving(false);
    }
  };

  return (
    <section className="space-y-3">
      <h2 className="text-base font-semibold text-gray-800">Document Storage</h2>

      <div className="flex items-center justify-between bg-white rounded-xl border border-gray-200 px-4 py-4">
        <div className="flex-1 pr-4">
          <p className="font-medium text-sm text-gray-800">Store documents after parsing</p>
          <p className="text-xs text-gray-500 mt-0.5">
            When enabled, your document files are kept encrypted in secure cloud storage.
            You can view them anytime. They are deleted 10 days after expiry.
          </p>
        </div>
        <Switch
          checked={pref === "full_storage"}
          onCheckedChange={handleToggle}
          disabled={isSaving}
          aria-label="Enable document storage"
        />
      </div>

      <div className="bg-amber-50 border border-amber-200 rounded-lg px-4 py-3">
        {pref === "parse_only" ? (
          <p className="text-xs text-amber-700">
            <strong>Parse-only mode (default):</strong> We extract the expiry date and immediately
            delete your document. No file is stored. Most secure option.
          </p>
        ) : (
          <p className="text-xs text-amber-700">
            <strong>Full storage enabled:</strong> Your documents are stored encrypted on Cloudflare R2.
            Access is via time-limited signed URLs (15 min). Files are auto-deleted 10 days after expiry.
          </p>
        )}
      </div>
    </section>
  );
}
```

### Signed URL Generation

[Source: architecture.md#7-external-apis (Cloudflare R2)]

In `src/lib/r2.ts` (stub from Story 1.3), implement `generateAccessUrl()`:

```typescript
import { S3Client, GetObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const s3Client = new S3Client({
  region: "auto",
  endpoint: `https://${process.env.R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID!,
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
  },
});

export async function generateAccessUrl(key: string, expiresInSeconds = 900): Promise<string> {
  const command = new GetObjectCommand({
    Bucket: process.env.R2_BUCKET_NAME!,
    Key: key,
  });
  return getSignedUrl(s3Client, command, { expiresIn: expiresInSeconds });
}
```

Install `@aws-sdk/s3-request-presigner` if not already present: `pnpm add @aws-sdk/s3-request-presigner`.

### GET /api/documents/[id]/url Route

[Source: architecture.md#5-api-specification]

Complete the stub created in Story 4.3:

```typescript
// src/app/api/documents/[id]/url/route.ts
export async function GET(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const doc = await documentService.getById(params.id, session.user.id);

    if (!doc.storageKey) {
      return NextResponse.json(
        { error: { code: "NOT_FOUND", message: "Document is not stored (parse-only mode)" } },
        { status: 404 }
      );
    }

    const signedUrl = await generateAccessUrl(doc.storageKey, 900);
    return NextResponse.json({ signedUrl });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### DB Schema Reference

[Source: architecture.md#9-database-schema]

The `users` table has:
```sql
document_storage_preference TEXT NOT NULL DEFAULT 'parse_only'
  CHECK (document_storage_preference IN ('parse_only', 'full_storage'))
```

The `documents` table has:
```sql
storage_url   TEXT,      -- null in parse_only mode; CDN/R2 URL in full_storage
storage_key   TEXT,      -- R2 object key; needed for deletion and signing
```

If `storageKey` column doesn't exist in the current schema, it must be added in `src/lib/db/schema.ts` as `storageKey: text("storage_key")`.
