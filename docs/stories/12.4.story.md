# Story 12.4 — Notification Preferences (Settings Wiring)

**Epic:** 12 — Profile & Settings
**Story:** 12.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 8.5 (notification preference components), Story 12.1 (settings page)

## Story

**As a** user,
**I want** all my settings on a single scrollable page,
**so that** I can find and change any preference without navigating between multiple sub-pages.

---

## Dev Notes

### Settings Page Architecture

[Source: architecture.md#12-unified-project-structure]

The settings page should be a hybrid RSC + client pattern:
- RSC (`src/app/(app)/profile/settings/page.tsx`) fetches current user settings server-side
- Passes as initial props to `SettingsView` (client component)

```tsx
// src/app/(app)/profile/settings/page.tsx (RSC)
export default async function SettingsPage() {
  const session = await getServerSession(authOptions);
  if (!session) redirect("/login");

  const user = await userService.getById(session.user.id);

  return (
    <SettingsView
      initialCurrency={user.currency ?? "INR"}
      initialNotificationWindowDays={user.notificationWindowDays ?? 30}
      initialEmailNotificationsEnabled={user.emailNotificationsEnabled ?? true}
      initialDocumentStoragePreference={user.documentStoragePreference ?? "store"}
    />
  );
}
```

### SettingsView Layout

[Source: front-end-spec.md#3-screen-by-screen-specifications]

```tsx
"use client";
export function SettingsView({ ... }: SettingsViewProps) {
  return (
    <div className="pb-20">
      <h1 className="text-xl font-bold text-gray-900 px-4 py-4">Settings</h1>

      {/* General */}
      <SectionHeader title="General" />
      <div className="bg-white divide-y divide-gray-100 border-y border-gray-200">
        <CurrencySetting currentCurrency={initialCurrency} />
      </div>

      {/* Notifications */}
      <SectionHeader title="Notifications" />
      <div className="bg-white divide-y divide-gray-100 border-y border-gray-200">
        <ExpiryWindowSetting value={initialNotificationWindowDays} />
        <PushNotificationSetting />
        <EmailNotificationSetting enabled={initialEmailNotificationsEnabled} />
      </div>

      {/* Privacy & Data */}
      <SectionHeader title="Privacy & Data" />
      <div className="bg-white divide-y divide-gray-100 border-y border-gray-200">
        <DocumentStorageSetting preference={initialDocumentStoragePreference} />
        <PrivacyPolicyLink />
        <DeleteStoredDocumentsAction />
        <DeleteAccountAction />
      </div>

      {/* Account */}
      <SectionHeader title="Account" />
      <div className="bg-white border-y border-gray-200">
        <SignOutButton />
      </div>
    </div>
  );
}

function SectionHeader({ title }: { title: string }) {
  return (
    <div className="px-4 py-2 mt-6">
      <p className="text-xs font-semibold text-gray-400 uppercase tracking-wider">{title}</p>
    </div>
  );
}
```

### userService.getById() — Full Profile Fields

[Source: architecture.md#6-components]

Ensure `userService.getById()` returns all settings fields needed:
- `currency`
- `notificationWindowDays`
- `emailNotificationsEnabled`
- `pushNotificationsEnabled`
- `documentStoragePreference`

These fields must exist in the `users` table. They are added in Stories 7.6, 8.3, 8.5, and 12.5. For this story, the agent must verify each field exists before using it. Use optional chaining + defaults in props if any are missing.
