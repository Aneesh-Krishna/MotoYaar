## Story

**As a** logged-in user,
**I want** to create a post in the community feed,
**so that** I can share tips, experiences, and content with other MotoYaar members.

2. - [ ] New post form at `src/app/(app)/community/new/page.tsx` OR as a bottom sheet — choose page for MVP

### DB Schema: posts Table

[Source: architecture.md#3-database-schema]

```sql
CREATE TABLE posts (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title           TEXT NOT NULL,
  description     TEXT NOT NULL,
  image_urls      TEXT[],                     -- array of R2 keys (max 2)
  link            TEXT,
  tags            TEXT[],                     -- predefined + custom tags
  is_hidden       BOOLEAN NOT NULL DEFAULT false,
  is_pinned       BOOLEAN NOT NULL DEFAULT false,
  pinned_at       TIMESTAMPTZ,
  edited          BOOLEAN NOT NULL DEFAULT false,
  edit_history    JSONB,                      -- [{title, description, editedAt}]
  edited_at       TIMESTAMPTZ,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_posts_user ON posts(user_id);
CREATE INDEX idx_posts_created_at ON posts(created_at DESC);
```

### Zod Schema

[Source: architecture.md#6-components]

Create `src/lib/validations/post.ts`:

```typescript
import { z } from "zod";

export const createPostSchema = z.object({
  title: z.string().min(1, "Title is required").max(200),
  description: z.string().min(1, "Description is required").max(1000),
  imageKeys: z.array(z.string()).max(2).optional().default([]),
  link: z.string().url("Must be a valid URL").optional().or(z.literal("")),
  tags: z.array(z.string().max(50)).max(10).optional().default([]),
});

export type CreatePostInput = z.infer<typeof createPostSchema>;
```

### Image Upload Flow

[Source: architecture.md#11-integrations]

1. User selects image(s) — validate type (JPG/PNG) and size (< 5 MB) client-side
2. Show local preview via `URL.createObjectURL(file)`
3. On form submit (or immediately on selection): `POST /api/upload/post-image` to get a pre-signed R2 URL
4. `PUT` file to pre-signed URL
5. Store the R2 key in `imageKeys` form field


Alternatively, if postId must be known: create the post record first with no images, then upload, then PATCH with image keys. Keep it simple — use temp UUID in key for MVP.

### Duplicate Submission Guard

[Source: architecture.md#6-components]

In `communityService.createPost()`:

```typescript
async createPost(userId: string, data: CreatePostInput): Promise<Post> {
  // 60s duplicate check
  const sixtySecondsAgo = new Date(Date.now() - 60_000);
  const existing = await db.query.posts.findFirst({
    where: and(
      eq(posts.userId, userId),
      eq(posts.title, data.title),
      eq(posts.description, data.description),
      gte(posts.createdAt, sixtySecondsAgo),
    ),
  });
  if (existing) {
    throw new ConflictError("Duplicate post submission");
  }

  const [post] = await db.insert(posts).values({
    userId,
    title: data.title,
    description: data.description,
    imageUrls: data.imageKeys ?? [],
    link: data.link || null,
    tags: data.tags ?? [],
  }).returning();

  return post;
}
```

`handleApiError` maps `ConflictError` to 409. The client catches 409 and shows toast "Your post was already submitted."

### POST /api/posts

[Source: architecture.md#5-api-specification]

Create `src/app/api/posts/route.ts`:

```typescript
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const body = await req.json();
    const data = createPostSchema.parse(body);
    const post = await communityService.createPost(session.user.id, data);
    return NextResponse.json(post, { status: 201 });
  } catch (error) {
    return handleApiError(error);
  }
}
```
