## Dev Notes

### The `?confirm=true` Requirement

[Source: architecture.md#5-api-specification]

The architecture spec explicitly requires `?confirm=true` on the DELETE endpoint:

> **DELETE `/api/trips/[id]` requires `?confirm=true` query param** — returns 400 without it (forces frontend to show warning modal first).

This prevents accidental deletions from direct API calls. The frontend only passes `?confirm=true` after the user clicks "Continue" in the `ConfirmModal`.

```typescript
export async function DELETE(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  const url = new URL(req.url);
  const confirmed = url.searchParams.get("confirm") === "true";

  if (!confirmed) {
    return NextResponse.json(
      { error: { code: "CONFIRMATION_REQUIRED", message: "Pass ?confirm=true to confirm deletion" } },
      { status: 400 }
    );
  }

  try {
    await tripService.delete(params.id, session.user.id);
    return new NextResponse(null, { status: 204 });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### tripService.delete()

[Source: architecture.md#6-components]

Add to `src/services/tripService.ts`:

```typescript
async delete(tripId: string, userId: string): Promise<void> {
  // 1. Ownership check
  const trip = await db.query.trips.findFirst({ where: eq(trips.id, tripId) });
  if (!trip) throw new NotFoundError("Trip not found");
  if (trip.userId !== userId) throw new ForbiddenError("You do not own this trip");

  // 2. Explicitly delete all linked expenses before deleting the trip
  // (Belt-and-suspenders: DB FK ON DELETE CASCADE will also handle this,
  //  but explicit delete ensures predictable order and logging)
  const linkedExpenses = await db.query.expenses.findMany({
    where: and(eq(expenses.tripId, tripId), eq(expenses.userId, userId)),
  });

  if (linkedExpenses.length > 0) {
    await db.delete(expenses).where(eq(expenses.tripId, tripId));
    logger.info({ tripId, count: linkedExpenses.length }, "Cascade-deleted trip expenses");
  }

  // 3. Delete the trip record
  await db.delete(trips).where(eq(trips.id, tripId));
}
```

CRITICAL: Per the coding standard called out in AC 5, only `tripService.delete()` should ever delete a trip record — no raw DB deletes on trips elsewhere in the codebase.

### Frontend Delete Flow

[Source: front-end-spec.md#5-component-library--ui-patterns]

In `TripKebabMenu` (Story 6.2), wire the Delete option:

```typescript
const [showDeleteModal, setShowDeleteModal] = useState(false);
const [isDeleting, setIsDeleting] = useState(false);
const router = useRouter();

const handleDelete = async () => {
  setIsDeleting(true);
  try {
    await deleteTrip(tripId); // passes ?confirm=true automatically
    toast.success("Trip deleted");
    router.push("/trips");
  } catch (error) {
    toast.error("Failed to delete trip. Try again.");
  } finally {
    setIsDeleting(false);
    setShowDeleteModal(false);
  }
};
```

```tsx
<ConfirmModal
  open={showDeleteModal}
  onClose={() => setShowDeleteModal(false)}
  onConfirm={handleDelete}
  title="Delete this trip?"
  description="Deleting this trip will also delete its associated expense entries. This cannot be undone. Continue?"
  confirmLabel="Delete Trip"
  isDestructive={true}
  isLoading={isDeleting}
/>
```

export async function deleteTrip(id: string): Promise<void> {
