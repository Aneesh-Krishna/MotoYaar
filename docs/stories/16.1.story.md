# Story 16.1 — Group Expense Session (Create & Join)

**Epic:** 16 — Groups & Club Features
**Story:** 16.1
**Status:** Ready for Review
**Created:** 2026-04-28
**Depends On:** Story 2.1 (auth)

---

## Story

**As a** rider going on a group trip,
**I want** to start a shared expense session and invite my fellow riders,
**so that** we can all track what we spent together and settle up later.

---

## Acceptance Criteria

1. - [x] A **"Start Group Expenses"** action is available from: (a) a trip detail page; (b) a standalone "New Group Session" entry in the Trips or Profile section
2. - [x] When started from a trip, the session is pre-linked to that trip (inheriting its title and currency defaults); the creator can then invite members manually by username
3. - [x] When started standalone, the creator can invite members by **MotoYaar username** — search auto-completes as they type
4. - [x] Invited members receive an in-app notification: *"[Creator] invited you to a group expense session for [trip title or 'a group trip']"* with [Join] / [Decline] options
5. - [x] Joining via notification adds the user to the session as a member
6. - [x] The creator can also share a **join link**: `motoyaar.app/group-expenses/join/[code]` — clicking the link while logged in auto-joins the session
7. - [x] A session has: a title (optional, defaults to trip title if linked), a currency (defaults to user's currency setting), and a list of members
8. - [x] The creator can remove a member before any expenses are logged; after expenses are logged, removal requires settling that member's balance to ₹0 first
9. - [x] A session's status is `active` until the creator archives it; archived sessions are read-only
10. - [x] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DB Schema

Key tables: `group_expense_sessions`, `group_expense_session_members`.

```sql
CREATE TABLE group_expense_sessions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title       TEXT,
  trip_id     UUID REFERENCES trips(id) ON DELETE SET NULL,
  created_by  UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  status      TEXT NOT NULL DEFAULT 'active'
                CHECK (status IN ('active', 'archived')),
  currency    TEXT NOT NULL DEFAULT 'INR',
  invite_code TEXT NOT NULL UNIQUE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE group_expense_session_members (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  UUID NOT NULL REFERENCES group_expense_sessions(id) ON DELETE CASCADE,
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  joined_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(session_id, user_id)
);
```

### Trip Link

When `trip_id` is provided, inherit `title` from the trip (if session title not specified) and use the creator's currency setting. No automatic member population — trips are single-user.

### Invite Code Generation

8-char alphanumeric — unique constraint at DB level; retry on collision.

### API Routes

```
POST   /api/group-expenses                          — create session
GET    /api/group-expenses/[id]                     — get session + members
PATCH  /api/group-expenses/[id]                     — update title / archive
POST   /api/group-expenses/[id]/members             — invite member by userId
DELETE /api/group-expenses/[id]/members/[userId]    — remove member
GET    /api/group-expenses/join/[code]              — resolve join link
POST   /api/group-expenses/join/[code]              — join via link
```
