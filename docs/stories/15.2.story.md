## Story

**As a** person looking to buy a used vehicle,
**I want** to look up a registration number and see the community-logged service history,
**so that** I can assess whether the vehicle has been properly maintained before I buy it.

---

## Acceptance Criteria

1. - [ ] A **"Vehicle History"** search page is accessible from the Community tab or the main search — user enters any registration number
2. - [ ] Results show a chronological list of service entries logged by MotoYaar users for that registration number
3. - [ ] Each history entry shows: service type, month + year (day hidden for privacy), service center name (if tagged), odometer at service (if logged)
4. - [ ] Owner identity (username, name) is **never** shown
5. - [ ] Expense amounts are **never** shown
6. - [ ] Entries soft-deleted by a user appear as: *"[Service type] — entry removed by owner · [Month Year]"* — the removal itself is visible
7. - [ ] If no history exists for a reg number: *"No service history found for [reg number]. Be the first to add your vehicle's history by logging expenses in MotoYaar."*
8. - [ ] User must be logged in to view history (no public unauthenticated access — reduces scraping)
9. - [ ] In user Settings, a toggle **"Include my vehicle data in community history lookups"** defaults to **on**. If turned off, that user's expense records are excluded from all history lookups.
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Soft-Delete on Expenses

Currently expenses support hard delete. For vehicle history integrity, expenses must support soft-delete:

```sql
ALTER TABLE expenses ADD COLUMN deleted_at TIMESTAMPTZ;
ALTER TABLE expenses ADD COLUMN deleted_by_owner BOOLEAN NOT NULL DEFAULT false;
```

When a user "deletes" an expense:
- For regular expenses (non-service, or service without history concern): hard delete (current behaviour unchanged)
- For service expenses: soft delete → set `deleted_at = now()`, `deleted_by_owner = true`

The vehicle history query returns soft-deleted service expenses with `deleted_by_owner = true` as tombstone entries (AC 6).

### History Query

```sql
SELECT
  e.reason,
  e.date,
  sc.name AS service_center_name,
  e.odometer_km,
  e.deleted_by_owner,
  e.deleted_at
FROM expenses e
LEFT JOIN service_centers sc ON e.service_center_id = sc.id
INNER JOIN vehicles v ON e.vehicle_id = v.id
WHERE
  v.registration_number = $1
  AND e.reason = 'Service'
  AND (
    -- Include if owner has opted in (default)
    EXISTS (SELECT 1 FROM users u WHERE u.id = v.user_id AND u.history_opt_out = false)
  )
ORDER BY e.date ASC;
```

### Privacy Opt-Out Column

```sql
ALTER TABLE users ADD COLUMN history_opt_out BOOLEAN NOT NULL DEFAULT false;
```

### API Route

```
GET /api/vehicle-history?reg=[REGISTRATION_NUMBER]
```

Auth required (AC 8). Returns only service expenses for that reg number across all owners who haven't opted out. Strips `user_id`, `vehicle_id`, `price`, `where_text`.

### Registration Number Normalisation

Strip spaces, hyphens, and convert to uppercase before query:
```typescript
const normalisedReg = reg.replace(/[\s-]/g, '').toUpperCase();
```

### Must-Fix Findings

#### F1 — AC1 FAIL: Vehicle History page is unreachable (no nav link)
The page exists at `/vehicle-history` but is linked from **nowhere**. The Community page, nav config (`src/config/nav.ts`), and no other surface reference it. AC1 explicitly requires accessibility "from the Community tab or the main search."

**Fix:** Add a link/card to `/vehicle-history` in the Community page or Profile page. A simple `Link` card in `src/app/(app)/community/page.tsx` matching the "Find a Mechanic" card pattern is the minimum required.

#### F2 — AC6 fragility: tombstone inclusion is implicit, not explicit
The history query correctly returns soft-deleted rows as tombstones because it has **no** `isNull(deletedAt)` exclusion. This is correct but fragile — if a developer adds the standard soft-delete exclusion filter (following the pattern from `expenseService.ts`), tombstones silently disappear with no failing tests.

**Fix:** Add a one-line comment in `vehicleHistoryService.getByReg` above the `where` clause:
```typescript
// Intentionally includes soft-deleted (deletedByOwner=true) rows as tombstones per AC6.
// Do NOT add isNull(deletedAt) here.
```
No logic change needed — comment only.
