# Story 3.6 — Delete Vehicle

**Epic:** 03 — Vehicle Management
**Story:** 3.6
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 3.4 (Vehicle Detail Page — delete accessible from kebab menu)

---

## Story

**As a** vehicle owner,
**I want** to delete a vehicle,
**so that** I can remove vehicles I no longer own and clean up my account.

---

## Acceptance Criteria

1. - [ ] Delete is accessible from the vehicle detail page kebab menu (owner only — viewers do not see Delete option)
2. - [ ] Before deleting, `ConfirmModal` is shown: "Deleting this vehicle will also delete all its documents, expenses, and trips. This cannot be undone. Continue?"
3. - [ ] Modal has two buttons: "Delete Vehicle" (destructive, red) and "Cancel"
4. - [ ] On confirm: `DELETE /api/vehicles/[id]` is called
5. - [ ] `vehicleService.delete(vehicleId, userId)` performs ownership check; cascade delete is handled by database FK `ON DELETE CASCADE` — service does not need to manually delete child records
6. - [ ] Vehicle image deleted from R2 if `imageUrl` exists (using `storageService.deleteFile(imageKey)`)
7. - [ ] All stored document files deleted from R2 if any documents have `storageUrl` set (query documents first, delete R2 files, then delete vehicle)
8. - [ ] Each user in `vehicle_access` for this vehicle receives an in-app notification: "A vehicle shared with you has been removed." (via `notificationService.create()`)
9. - [ ] On confirm and success: redirect to `/garage`
10. - [ ] Cancel closes the modal without any action
11. - [ ] `ConfirmModal` component lives at `src/components/ui/ConfirmModal.tsx` (reusable for other delete flows)
12. - [ ] `pnpm typecheck` and `pnpm lint` pass

### vehicleService.delete()

[Source: architecture.md#6-components]

Add to `src/services/vehicleService.ts`:

```typescript
async delete(vehicleId: string, userId: string): Promise<void> {
  // 1. Ownership check
  const vehicle = await db.query.vehicles.findFirst({
    where: eq(vehicles.id, vehicleId),
  });
  if (!vehicle) throw new NotFoundError("Vehicle not found");
  if (vehicle.userId !== userId) throw new ForbiddenError("Only the vehicle owner can delete this vehicle");

  // 2. Get all documents with stored R2 files before deleting
  const storedDocs = await db.query.documents.findMany({
    where: and(
      eq(documents.vehicleId, vehicleId),
      isNotNull(documents.storageUrl)
    ),
  });

  // 3. Get all viewers (for notification)
  const viewers = await db.query.vehicleAccess.findMany({
    where: eq(vehicleAccess.vehicleId, vehicleId),
  });

  // 4. Delete vehicle — FK ON DELETE CASCADE handles child records (documents, expenses, trips, vehicle_access)
  await db.delete(vehicles).where(eq(vehicles.id, vehicleId));

  // 5. Delete R2 files (after DB delete — order matters; DB is source of truth)
  if (vehicle.imageUrl && vehicle.imageKey) {
    await storageService.deleteFile(vehicle.imageKey).catch((e) =>
      logger.error({ error: e, vehicleId }, "Failed to delete vehicle image from R2")
    );
  }
  for (const doc of storedDocs) {
    if (doc.storageKey) {
      await storageService.deleteFile(doc.storageKey).catch((e) =>
        logger.error({ error: e, docId: doc.id }, "Failed to delete document from R2")
      );
    }
  }

  // 6. Notify viewers
  for (const viewer of viewers) {
    await notificationService.create({
      userId: viewer.viewerId,
      type: "vehicle_removed",
      message: "A vehicle shared with you has been removed.",
    }).catch((e) =>
      logger.error({ error: e, viewerId: viewer.viewerId }, "Failed to send vehicle removal notification")
    );
  }
}
```

Key notes:
- DB cascade (`ON DELETE CASCADE` on `vehicles.id`) handles deleting all child `documents`, `expenses`, `trips`, `vehicle_access` rows automatically — the service does NOT need explicit deletes for those tables.
- R2 file cleanup happens AFTER the DB delete. Use `.catch()` on each R2 operation so R2 failures do not break the delete flow (log them).
- Notification failures are also caught and logged — they must not roll back the delete.

### Frontend Delete Flow

[Source: front-end-spec.md#5-component-library--ui-patterns]

In the vehicle detail page, the kebab menu opens the `ConfirmModal` when Delete is clicked.

The delete handler inside `VehicleKebabMenu` (or a parent component):

```typescript
const [showDeleteModal, setShowDeleteModal] = useState(false);
const [isDeleting, setIsDeleting] = useState(false);
const router = useRouter();

const handleDelete = async () => {
  setIsDeleting(true);
  try {
    await deleteVehicle(vehicleId); // vehicleApi.deleteVehicle(id)
    toast.success("Vehicle deleted");
    router.push("/garage");
  } catch (error) {
    if (error instanceof ApiError) {
      toast.error(error.message);
    } else {
      toast.error("Failed to delete vehicle. Try again.");
    }
  } finally {
    setIsDeleting(false);
    setShowDeleteModal(false);
  }
};
```
