# Story 4.6 — Driver's License Management

## Story

**As a** user,
**I want** to upload and track my Driver's License expiry,
**so that** I get alerted before my DL expires just like my vehicle documents.

---

## Acceptance Criteria

1. - [ ] DL upload is accessible from the Profile page — a "Driver's License" section with a `StatusBadge` and "Upload" or "Update" CTA
2. - [ ] Same AI parse flow as Story 4.1 — same `BottomSheet` + parse + confirm/manual-entry screens
3. - [ ] DL document stored with `vehicleId = null`, `userId = currentUser.id`, `type = "DL"`
4. - [ ] Separate API endpoints: `POST /api/users/me/documents/parse` (AI parse) and `POST /api/users/me/documents` (save) — distinct from vehicle document endpoints
5. - [ ] Profile page shows the DL section: DL type icon + expiry date + `StatusBadge`; if no DL uploaded, shows "Not uploaded" with "Upload" CTA
6. - [ ] Edit and delete available from the Profile page DL section
7. - [ ] DL is included in the daily cron notification check (Story 8.1) — the cron queries `documents WHERE vehicle_id IS NULL AND type = 'DL'` — this story ensures the DB record is written correctly
8. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DL Document Record

[Source: architecture.md#4-data-models (Document)]

The `Document` type supports `vehicleId?: string` — nullable. For DL:

```typescript
{
  vehicleId: null,          // no vehicle — belongs to user directly
  userId: currentUserId,
  type: "DL",
  expiryDate: "...",
  parseStatus: "parsed" | "manual" | "incomplete",
  storageUrl: null | string,
  storageKey: null | string,
}
```

The DB `documents` table has `vehicle_id UUID REFERENCES vehicles(id) ON DELETE CASCADE` — nullable. Records with `vehicle_id = NULL` are user-level documents (DL).

### API Routes: /api/users/me/documents

[Source: architecture.md#5-api-specification]

**Parse route:**
```typescript
// src/app/api/users/me/documents/parse/route.ts
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const formData = await req.formData();
    const file = formData.get("file") as File;

    if (!file) return NextResponse.json({ error: "No file provided" }, { status: 400 });
    if (file.size > 10 * 1024 * 1024) {
      return NextResponse.json({ error: "File too large (max 10MB)" }, { status: 400 });
    }

    const tempKey = `${session.user.id}/documents/temp/dl-${crypto.randomUUID()}.jpg`;
    const fileBuffer = await file.arrayBuffer();
    await storageService.uploadFile(tempKey, Buffer.from(fileBuffer), file.type);

    const result = await parseDocument(fileBuffer, file.type);

    return NextResponse.json({
      extractedExpiryDate: result.expiryDate ?? null,
      documentType: "DL",
      confidence: result.confidence,
      parseStatus: result.expiryDate ? "parsed" : "failed",
      tempR2Key: tempKey,
    });
  } catch (error) {
    return handleApiError(error);
  }
}
```

```typescript
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const body = await req.json();
    const data = createDocumentSchema.parse(body);
    // DL-specific: vehicleId = null
    const doc = await documentService.createUserDocument(session.user.id, data);
    return NextResponse.json(doc, { status: 201 });
  } catch (error) {
    return handleApiError(error);
  }
}

export async function GET(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const docs = await documentService.listUserDocuments(session.user.id);
    return NextResponse.json(docs);
  } catch (error) {
    return handleApiError(error);
  }
}
```

### documentService Methods for DL

[Source: architecture.md#6-components]

Add to `src/services/documentService.ts`:

```typescript
async createUserDocument(userId: string, data: CreateDocumentInput): Promise<Document> {
  // Same logic as create() but vehicleId = null
  const user = await db.query.users.findFirst({ where: eq(users.id, userId) });
  const pref = user?.documentStoragePreference ?? "parse_only";

  let storageUrl: string | null = null;
  let storageKey: string | null = null;

  if (data.tempR2Key) {
    if (pref === "full_storage") {
      const permanentKey = `${userId}/documents/dl/${crypto.randomUUID()}.jpg`;
      await storageService.copyFile(data.tempR2Key, permanentKey);
      await storageService.deleteFile(data.tempR2Key);
      storageKey = permanentKey;
    } else {
      await storageService.deleteFile(data.tempR2Key).catch((e) =>
        logger.error({ error: e }, "Failed to delete DL temp R2 file")
      );
    }
  }

  const [doc] = await db.insert(documents).values({
    vehicleId: null,   // ← critical: no vehicle linkage for DL
    userId,
    type: "DL",
    label: data.label,
    expiryDate: data.expiryDate ? new Date(data.expiryDate) : null,
    storageUrl,
    storageKey,
    parseStatus: data.parseStatus,
    status: computeDocumentStatus(data.expiryDate ?? null, user?.notificationWindowDays ?? 30),
  }).returning();

  return doc;
}

async listUserDocuments(userId: string): Promise<Document[]> {
  const user = await db.query.users.findFirst({ where: eq(users.id, userId) });
  const windowDays = user?.notificationWindowDays ?? 30;

  const docs = await db.query.documents.findMany({
    where: and(
      eq(documents.userId, userId),
      isNull(documents.vehicleId)
    ),
  });

  return docs.map((doc) => ({
    ...doc,
    status: computeDocumentStatus(
      doc.expiryDate ? doc.expiryDate.toISOString().split("T")[0] : null,
      windowDays
    ),
  }));
}
```

### DocumentUpload Mode Prop

[Source: architecture.md#6-components]

The `DocumentUpload` component (Story 4.1) needs a `mode` prop to switch between vehicle and DL endpoints:

```typescript
interface DocumentUploadProps {
  vehicleId?: string;   // vehicle document mode
  mode?: "vehicle" | "dl";  // default: "vehicle"
  onComplete: () => void;
}
```

When `mode === "dl"`:
- Parse endpoint: `POST /api/users/me/documents/parse`
- Save endpoint: `POST /api/users/me/documents`
- Document type is locked to `"DL"` (not user-selectable)

### Cron Query Compatibility

[Source: architecture.md#8-core-workflows (8.4 Daily Cron)]

The daily cron in Story 8.1 queries:
```sql
SELECT d.*, u.notification_window_days
FROM documents d
JOIN users u ON u.id = d.user_id
WHERE d.expiry_date IS NOT NULL
```

This query picks up ALL documents including DL (where `vehicle_id IS NULL`). The DL record written by `createUserDocument()` is automatically included — no special cron handling needed in this story.
