# Story 16.5 — Community Route Library

**Epic:** 16 — Groups & Club Features
**Story:** 16.5
**Status:** Draft
**Created:** 2026-04-28
**Depends On:** Story 13.2 (GPS Route Recording — `trip_routes` records must exist), Story 14.1 (Public Trip Route Sharing — reuses route thumbnail infrastructure)

---

## Story

**As a** rider planning a trip,
**I want** to browse and search GPS routes recorded by other MotoYaar users,
**so that** I can find a proven route for my destination instead of guessing from a map.

---

## Acceptance Criteria

1. - [ ] A **"Route Library"** section is accessible from the Community tab
2. - [ ] Any user can **publish** a completed live trip's route to the library from the trip detail page ("Publish to Route Library" option alongside "Share Route")
3. - [ ] Publishing requires the trip to have a saved GPS route (`has_live_route = true`)
4. - [ ] Each published route shows: route name (defaults to trip title, editable on publish), distance (km), estimated duration (from route data), start region (auto-detected from waypoints), tags (editable, predefined: Highways, Mountain, Offroad, City, Coastal, Scenic), and a map thumbnail
5. - [ ] The Route Library is **searchable** by region name (free text) and filterable by tags
6. - [ ] Any logged-in user can **download** a published route as a GPX file
7. - [ ] Any logged-in user can **save** a route to their personal "Saved Routes" list (bookmark mechanic)
8. - [ ] Routes are sorted by: Most Recent (default) | Most Downloaded | Most Saved
9. - [ ] The publisher's username is shown on the route card with a link to their profile
10. - [ ] The publisher can un-publish their route at any time — it is removed from the library but the underlying `trip_routes` record is preserved
11. - [ ] Route library is browsable by non-logged-in users (guest access for discovery); download and save require login
12. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DB Schema

```sql
CREATE TABLE published_routes (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  trip_route_id   UUID NOT NULL REFERENCES trip_routes(id) ON DELETE CASCADE,
  user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name            TEXT NOT NULL,
  tags            TEXT[] NOT NULL DEFAULT '{}',
  start_region    TEXT,                          -- auto-detected from first waypoint (reverse geocode or bounding box lookup)
  distance_km     NUMERIC(8,3),                  -- copied from trip_routes.distance_km
  download_count  INTEGER NOT NULL DEFAULT 0,
  save_count      INTEGER NOT NULL DEFAULT 0,
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(trip_route_id)                          -- one published entry per route
);

CREATE TABLE saved_routes (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id             UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  published_route_id  UUID NOT NULL REFERENCES published_routes(id) ON DELETE CASCADE,
  saved_at            TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(user_id, published_route_id)
);

CREATE INDEX idx_published_routes_user_id ON published_routes(user_id);
CREATE INDEX idx_published_routes_tags ON published_routes USING GIN(tags);
CREATE INDEX idx_published_routes_region ON published_routes(start_region);
```

### GPX Export

GPX is a standard XML format for GPS tracks. Generate server-side from waypoints:

```typescript
function generateGPX(route: TripRoute, name: string): string {
  const points = route.waypoints.map(wp =>
    `<trkpt lat="${wp.lat}" lon="${wp.lng}">
      <time>${new Date(wp.timestamp).toISOString()}</time>
      ${wp.altitude ? `<ele>${wp.altitude}</ele>` : ''}
    </trkpt>`
  ).join('\n');

  return `<?xml version="1.0" encoding="UTF-8"?>
<gpx version="1.1" creator="MotoYaar">
  <trk><name>${name}</name><trkseg>
    ${points}
  </trkseg></trk>
</gpx>`;
}
```

Served via `GET /api/routes/[id]/gpx` with `Content-Type: application/gpx+xml` and `Content-Disposition: attachment; filename="route.gpx"`.

### Start Region Detection

On publish, detect the start region from the first waypoint using a simple lat/lng → Indian state/region lookup (a static bounding box table for major Indian states is sufficient — no external geocoding API needed):

```typescript
const INDIAN_REGIONS = [
  { name: 'Ladakh',         minLat: 32.5, maxLat: 36.0, minLng: 75.0, maxLng: 80.0 },
  { name: 'Himachal Pradesh', minLat: 30.5, maxLat: 33.5, minLng: 75.5, maxLng: 79.5 },
  // ... all Indian states
];
```

First match wins. Falls back to `null` if no match — user can manually edit the region on publish.

### API Routes

```
GET    /api/routes                    — list/search published routes (public)
POST   /api/routes                    — publish a route (auth required)
GET    /api/routes/[id]               — route detail (public)
DELETE /api/routes/[id]               — un-publish (owner only)
GET    /api/routes/[id]/gpx           — download GPX (auth required)
POST   /api/routes/[id]/save          — save to personal list (auth required)
DELETE /api/routes/[id]/save          — unsave (auth required)
GET    /api/users/me/saved-routes     — user's saved routes
```
