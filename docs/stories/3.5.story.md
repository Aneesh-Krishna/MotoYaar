# Story 3.5 — Edit Vehicle

**Epic:** 03 — Vehicle Management

**Status:** Draft

## Story

**As a** vehicle owner,
**I want** to edit my vehicle's details,
**so that** I can correct mistakes or update information over time.

---

## Acceptance Criteria

1. - [ ] Route: `src/app/(app)/garage/[id]/edit/page.tsx` — accessible from the vehicle detail kebab menu (owner only)
2. - [ ] Only the vehicle owner can access this page; non-owners (viewers) receive a 403 redirect to `/garage/[id]`
3. - [ ] Single-page form (not a wizard) pre-populated with all current vehicle data
4. - [ ] Form fields: Name, Type, Company, Model, Variant, Color, Registration Number, Purchase Date, Previous Owners, Vehicle Image
5. - [ ] Vehicle image: can be updated (new upload via pre-signed R2 URL, same pattern as Story 3.3) or kept as-is; existing image thumbnail shown
6. - [ ] On save: `PATCH /api/vehicles/[id]` called; registration number uniqueness checked server-side if reg number changed
7. - [ ] On successful save: redirect to `/garage/[id]`
8. - [ ] Cancel button: returns to `/garage/[id]` without saving; no confirmation required
9. - [ ] Zod schema: `updateVehicleSchema` from `src/lib/validations/vehicle.ts` (already defined in Story 3.3 — all fields optional except registration number stays required if provided)
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Route and Page Structure

[Source: architecture.md#12-unified-project-structure]

`src/app/(app)/garage/[id]/edit/page.tsx` exists as a stub from Story 1.1. Read it before editing.

This page is a client component — it manages form state and needs the vehicle data pre-fetched.

```tsx
// src/app/(app)/garage/[id]/edit/page.tsx (RSC wrapper)
import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";
import { vehicleService } from "@/services/vehicleService";
import { redirect, notFound } from "next/navigation";
import { ForbiddenError, NotFoundError } from "@/lib/errors";
import { EditVehicleForm } from "@/components/vehicles/EditVehicleForm";

export default async function EditVehiclePage({ params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) redirect("/login");

  try {
    const vehicle = await vehicleService.getByIdOwnerOnly(params.id, session.user.id);
    return <EditVehicleForm vehicle={vehicle} />;
  } catch (error) {
    if (error instanceof ForbiddenError) redirect(`/garage/${params.id}`);
    if (error instanceof NotFoundError) notFound();
    throw error;
  }
}
```

The page is RSC; `EditVehicleForm` is a `"use client"` component.

### vehicleService.getByIdOwnerOnly()

[Source: architecture.md#6-components]

Add to `src/services/vehicleService.ts`:

```typescript
async getByIdOwnerOnly(vehicleId: string, userId: string): Promise<Vehicle> {
  const vehicle = await db.query.vehicles.findFirst({
    where: eq(vehicles.id, vehicleId),
  });

  if (!vehicle) throw new NotFoundError("Vehicle not found");
  if (vehicle.userId !== userId) throw new ForbiddenError("Only the vehicle owner can edit this vehicle");

  return vehicle;
}
```

This is distinct from `getWithAccessCheck` — viewers have read access (Story 3.4) but not write access.

### Image Update Flow

[Source: architecture.md#7-external-apis]

When the user taps "Change Photo":
1. Open file picker (same constraints as Story 3.3: `image/jpeg,image/png,image/webp`, max 5MB)
2. Call `POST /api/uploads/vehicle-image` → get pre-signed PUT URL + new key
3. PUT file directly to R2
4. If user had a previous image: call `DELETE /api/uploads/vehicle-image?key={oldImageKey}` to clean up
5. Update form field `imageUrl` and `imageKey` with new values

Show image thumbnail at 80×80px with a "Change" button overlay, or a placeholder box if no existing image.

export async function PATCH(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const body = await req.json();
    const data = updateVehicleSchema.parse(body);
    const vehicle = await vehicleService.update(params.id, session.user.id, data);
    return NextResponse.json(vehicle);
  } catch (error) {
    return handleApiError(error);
  }
}

### vehicleService.update()

[Source: architecture.md#6-components]

Add to `src/services/vehicleService.ts`:

```typescript
async update(vehicleId: string, userId: string, data: UpdateVehicleInput): Promise<Vehicle> {
  // Ownership check
  const vehicle = await db.query.vehicles.findFirst({
    where: eq(vehicles.id, vehicleId),
  });
  if (!vehicle) throw new NotFoundError("Vehicle not found");
  if (vehicle.userId !== userId) throw new ForbiddenError("Only the vehicle owner can edit this vehicle");

  // Registration number uniqueness check (only if reg number is being changed)
  if (data.registrationNumber && data.registrationNumber.toUpperCase() !== vehicle.registrationNumber) {
    const duplicate = await db.query.vehicles.findFirst({
      where: and(
        eq(vehicles.userId, userId),
        eq(vehicles.registrationNumber, data.registrationNumber.toUpperCase())
      ),
    });
    if (duplicate) throw new ConflictError("You already have a vehicle with this registration number");
  }

  const [updated] = await db.update(vehicles)
    .set({
      ...data,
      registrationNumber: data.registrationNumber
        ? data.registrationNumber.toUpperCase()
        : undefined,
      updatedAt: new Date(),
    })
    .where(eq(vehicles.id, vehicleId))
    .returning();

  return updated;
}
```

### Error Handling

[Source: architecture.md#18-error-handling-strategy]

- `409 Conflict` (duplicate reg number) → inline error below Registration Number field: "You already have a vehicle with this registration number"
- `403 Forbidden` → this should be caught by the page-level RSC guard; show a toast if it occurs on form submit
- `422 Validation` → React Hook Form + Zod handles client-side; server returns 422 as fallback
- Network error → Sonner toast: "Failed to update vehicle. Try again."
