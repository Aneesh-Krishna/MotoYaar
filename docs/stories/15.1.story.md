## Story

**As a** vehicle owner,
**I want** to tag and rate the service center where I got my vehicle serviced,
**so that** other MotoYaar users can find reliable mechanics near them.

---

## Acceptance Criteria

1. - [ ] On the expense form, when `reason = "Service"`, the existing **"Where?"** free-text field is replaced with a **searchable service center picker**
2. - [ ] Typing in the picker searches existing `service_centers` records by name + city
3. - [ ] If no match found, an option *"Add '[typed name]' as a new service center"* is shown — tapping it creates a new `service_centers` record inline with name (required), city (required), pincode (optional)
4. - [ ] Tagging is optional — the "Where?" field can still be left blank
5. - [ ] After saving a service expense with a tagged service center, a **dismissible prompt** appears: *"Rate [Service Center Name]?"* with [Rate Now] and [Skip] options
6. - [ ] Rating form: star picker (1–5, required) + optional review text (max 100 chars). Submit creates a `service_center_reviews` record.
7. - [ ] A user can only submit one review per service center (UNIQUE constraint). Submitting again updates the existing review.
8. - [ ] Service center profile page at `/service-centers/[id]`: shows name, city, average star rating, total review count, last 10 reviews (star + text + date; no reviewer username shown for privacy)
9. - [ ] Service centers are searchable from a **"Find a Mechanic"** entry point in the Community tab or Profile
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DB Schema

```sql
CREATE TABLE service_centers (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL,
  city        TEXT NOT NULL,
  pincode     TEXT,
  created_by  UUID REFERENCES users(id) ON DELETE SET NULL,
  avg_rating  NUMERIC(3,2),      -- denormalised; updated on review insert/update
  review_count INTEGER NOT NULL DEFAULT 0,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_service_centers_city ON service_centers(city);
CREATE INDEX idx_service_centers_name ON service_centers USING GIN(to_tsvector('english', name));

CREATE TABLE service_center_reviews (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  service_center_id UUID NOT NULL REFERENCES service_centers(id) ON DELETE CASCADE,
  user_id           UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  rating            INTEGER NOT NULL CHECK (rating BETWEEN 1 AND 5),
  review_text       TEXT CHECK (char_length(review_text) <= 100),
  created_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(service_center_id, user_id)
);
```

`service_centers.avg_rating` and `review_count` are updated via a trigger or in the service layer on each review insert/update to avoid expensive aggregation on read.

### Service Center Search API

```
GET /api/service-centers?q=sharma+auto&city=Pune
```
Uses `ILIKE '%query%'` on name + city. Returns top 10 matches. Light-weight — no full-text index needed at MVP scale.

### "Add New" Inline Flow

When user taps "Add '[name]' as a new service center":
- A small inline form appears below the search field (not a separate page)
- Fields: City (required), Pincode (optional)
- Name is pre-filled from the search text
- On save: `POST /api/service-centers` → returns new record → auto-selected in the picker

### Average Rating Update

In `serviceCenterService.upsertReview()`:
```typescript
// After upsert, recalculate avg + count
const { avg, count } = await db
  .select({ avg: avg(reviews.rating), count: count() })
  .from(reviews)
  .where(eq(reviews.serviceCenterId, id));

await db.update(serviceCenters)
  .set({ avgRating: avg, reviewCount: count })
  .where(eq(serviceCenters.id, id));
```
