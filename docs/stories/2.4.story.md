# Story 2.4 — Re-access Walkthrough from Settings

**Epic:** 02 — Authentication & Onboarding
**Story:** 2.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 2.3 (Walkthrough modal component)

---

## Story

**As a** user who skipped or wants to revisit the walkthrough,
**I want** to access it again from Settings,
**so that** I can re-orient myself at any time.

---

## Acceptance Criteria

1. - [ ] "App tour" or "How it works" option visible in Settings page
2. - [ ] Tapping it opens the same 5-card walkthrough modal (reuses the `WalkthroughModal` component from Story 2.3)
3. - [ ] Works identically to the first-time flow (Back / Skip / Get started navigation)
4. - [ ] On completion or skip: closes modal and returns to Settings (does NOT redirect to Dashboard)

### `AppTourButton` Component

The button should open the walkthrough modal **inline** (without navigating away from Settings). Two implementation options:

**Option A (Recommended):** Use the `WalkthroughModal` component from Story 2.3 as a client-side modal overlay triggered by a button on the settings page.

```tsx
"use client";
import { useState } from "react";
import { WalkthroughModal } from "@/components/onboarding/WalkthroughModal";

export function AppTourButton() {
  const [open, setOpen] = useState(false);
  return (
    <>
      <button onClick={() => setOpen(true)}>App tour</button>
      {open && (
        <WalkthroughModal
          onComplete={() => setOpen(false)}
          onSkip={() => setOpen(false)}
          // Don't save walkthroughSeen when re-opening from settings
          skipSave={true}
        />
      )}
    </>
  );
}
```

**Option B:** Navigate to `/walkthrough?from=settings` and redirect back to Settings on close.

> Use Option A — no page navigation, smoother UX.

---

### WalkthroughModal Props Update

[Source: Story 2.3]

The `WalkthroughModal` component needs to support being re-opened from Settings without re-saving `walkthroughSeen` to the DB (since it was already true). Add a `skipSave?: boolean` prop:

- `skipSave = false` (default): saves `walkthroughSeen = true` and redirects to Dashboard on complete/skip
- `skipSave = true`: closes modal and calls `onComplete`/`onSkip` callback without saving or redirecting
