# Story 5.3 — Edit Expense

**Epic:** 05 — Expense Tracking
**Story:** 5.3
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 5.2 (Expense List — edit initiated by tapping a row)

---

## Story

**As a** user,
**I want** to edit an expense I've logged,
**so that** I can fix mistakes or add missing information.

---

## Acceptance Criteria

1. - [ ] Tapping an expense row in the list opens a pre-populated edit form in a `BottomSheet`
2. - [ ] All fields editable: Price, Date, Reason (see AC 3), Where, Comments, Receipt (stub — Story 5.5)
3. - [ ] Trip-linked expenses (`expense.tripId !== null`): form shows a read-only indicator "This expense is linked to a trip. Edit it from the trip." with a link to the trip detail page; all fields are read-only
4. - [ ] Reason field: cannot be changed to or from "Trip" — if `reason === "Trip"`, it is display-only (not a chip selector); if reason is not "Trip", the "Trip" chip is absent from the selector
5. - [ ] `PATCH /api/expenses/[id]` called on save
6. - [ ] `expenseService.update(expenseId, userId, data)` — ownership check included
7. - [ ] On save: expense updated; BottomSheet closed; expense list refreshes via `router.refresh()`
8. - [ ] `pnpm typecheck` and `pnpm lint` pass

### expenseService.update()

[Source: architecture.md#6-components]

Add to `src/services/expenseService.ts`:

```typescript
async update(expenseId: string, userId: string, data: UpdateExpenseInput): Promise<Expense> {
  const expense = await db.query.expenses.findFirst({
    where: eq(expenses.id, expenseId),
  });

  if (!expense) throw new NotFoundError("Expense not found");
  if (expense.userId !== userId) throw new ForbiddenError("You do not own this expense");

  // Block edits to trip-linked expenses
  if (expense.tripId) {
    throw new ForbiddenError("Trip-linked expenses can only be edited via the trip");
  }

  const [updated] = await db.update(expenses)
    .set({
      price: data.price,
      date: data.date ? new Date(data.date) : undefined,
      reason: data.reason,
      whereText: data.whereText,
      comment: data.comment,
    })
    .where(eq(expenses.id, expenseId))
    .returning();

  return updated;
}
```
