# Story 9.7 — Tag Filtering & Search

**Epic:** 09 — Community
**Story:** 9.7
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 9.2 (feed with tag filter strip already exists)

---

## Story

**As a** user,
**I want** to filter the feed by tag and search for posts by keyword,
**so that** I can quickly find content relevant to my interests.

---

## Acceptance Criteria

1. - [ ] Tapping a tag chip on the feed filters to posts with that tag; active tag shown with highlighted state and "×" dismiss chip above feed
2. - [ ] Tag filtering passes `?tag=` query param to `GET /api/posts`
3. - [ ] Search bar in Community tab: visible above the feed
4. - [ ] `GET /api/posts?q=searchterm` — `WHERE title ILIKE '%q%' OR description ILIKE '%q%'`
5. - [ ] Search results rendered in same feed format using `PostCard`; sorted by relevance then recency (posts matching title ranked above description matches; same level sorted by `created_at DESC`)
6. - [ ] Empty state when no results: "No posts found for '[query]'"
7. - [ ] Search debounced 300 ms before API call is made
8. - [ ] Active tag and search query reflected in URL query params (e.g. `?tag=Bikes&q=helmet`) for shareable/back-navigable URLs
9. - [ ] `communityService.listPosts()` accepts optional `q` parameter
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

### communityService.listPosts() — q Parameter

[Source: architecture.md#6-components]

Extend the signature and DB query:

```typescript
async listPosts(
  sort: "trending" | "newest",
  page: number,
  tag?: string,
  userId?: string,
  q?: string
): Promise<{ posts: FeedPost[]; hasMore: boolean }> {
  const whereConditions = [
    eq(posts.isHidden, false),
    tag ? sql`${tag} = ANY(${posts.tags})` : undefined,
    q
      ? or(
          ilike(posts.title, `%${q}%`),
          ilike(posts.description, `%${q}%`)
        )
      : undefined,
  ].filter(Boolean);

  const rawPosts = await db.query.posts.findMany({
    where: and(...whereConditions),
    // ... rest of query
  });

  // Sort: when q is present, title matches first
  if (q) {
    const qLower = q.toLowerCase();
    return rawPosts.sort((a, b) => {
      const aTitle = a.title.toLowerCase().includes(qLower);
      const bTitle = b.title.toLowerCase().includes(qLower);
      if (aTitle && !bTitle) return -1;
      if (!aTitle && bTitle) return 1;
      return b.createdAt.getTime() - a.createdAt.getTime();
    });
  }

  // Normal sort (trending / newest)
  // ...
}
```

### Search Bar UI

[Source: front-end-spec.md#5-component-library--ui-patterns]

Add a search input above the sort toggle in the community page:

```tsx
// In community page client component
const [searchQuery, setSearchQuery] = useState("");
const [debouncedQuery] = useDebounce(searchQuery, 300);

  resetAndFetch({ q: debouncedQuery || undefined });
```

### URL State Synchronisation

[Source: architecture.md#5-api-specification]

Use Next.js `useRouter` and `useSearchParams` to keep URL in sync with filter state:

```typescript
const router = useRouter();
const params = useSearchParams();

// On filter change:
const updateUrl = (updates: Record<string, string | undefined>) => {
  const newParams = new URLSearchParams(params.toString());
  Object.entries(updates).forEach(([key, value]) => {
    if (value) newParams.set(key, value);
    else newParams.delete(key);
  });
  router.replace(`/community?${newParams.toString()}`, { scroll: false });
};
```

Initialise state from URL params on mount:

```typescript
const [activeTag, setActiveTag] = useState(params.get("tag") ?? undefined);
const [searchQuery, setSearchQuery] = useState(params.get("q") ?? "");
```
