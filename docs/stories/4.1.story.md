## Story

**As a** vehicle owner,
**I want** to upload a vehicle document and have the expiry date extracted automatically,
**so that** I don't have to read and manually enter dates from my documents.

---

## Acceptance Criteria

1. - [ ] Document upload is accessible from: Vehicle Detail → Documents tab → "Add document" CTA; and from Add Vehicle Step 5
2. - [ ] Supported document types (user selects): RC, Insurance, PUC, Other (user-labeled)
3. - [ ] Supported file types: JPG, PNG, PDF; max file size: 10MB; client-side validation before upload
4. - [ ] Upload flow has 4 screens in a `BottomSheet`: (1) Upload screen with file picker + camera capture option, (2) Parsing loading state "Reading your document..." with animation, (3) Confirmation screen with extracted expiry date (editable), (4) Save confirmation
5. - [ ] If AI returns no expiry date: skip to manual entry (Story 4.2 flow — date picker)
6. - [ ] Server flow: `POST /api/vehicles/[id]/documents/parse` → upload to R2 (temp key) → call `parseDocument()` from `src/lib/anthropic.ts` → return `{ expiryDate, parseStatus }`
7. - [ ] Client confirms: `POST /api/vehicles/[id]/documents` with `{ type, expiryDate, parseStatus, confirmed: true }` saves the document record
8. - [ ] Storage preference: if `user.documentStoragePreference === "parse_only"` → delete R2 file after parsing; if `"full_storage"` → keep file, set `storageUrl` on document record
9. - [ ] Document record created with: `type`, `expiryDate`, `storageUrl` (null if parse_only), `parseStatus` (`"parsed"` | `"manual"` | `"incomplete"`)
10. - [ ] PDF handling: convert first page to an image client-side before sending to server (avoids server-side PDF processing); implementation note: use `pdfjs-dist` in the browser
11. - [ ] `BottomSheet` component used for the upload flow on mobile
12. - [ ] `documentService.create()` in `src/services/documentService.ts` handles DB insert
13. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Upload Flow Architecture

[Source: architecture.md#8-core-workflows (8.2 Document Upload + AI Parsing)]

The full two-step flow:

**Step 1 — Parse:**
```
Client selects file → (PDF? convert to PNG first) →
POST /api/vehicles/[id]/documents/parse (multipart: file + docType) →
Server: upload to R2 with temp key → call parseDocument() →
Return { extractedExpiryDate, documentType, confidence, parseStatus }
```

**Step 2 — Save:**
```
User reviews/edits date on Confirmation screen →
POST /api/vehicles/[id]/documents { type, expiryDate, parseStatus, tempR2Key, confirmed: true } →
Server: if parse_only → delete R2 temp file; if full_storage → move/keep R2 file →
INSERT into documents → Return 201 { document }
```

The temp R2 key from step 1 is passed back in step 2 so the server knows which R2 file to keep or delete.

### PDF to Image Conversion (Client-Side)

[Source: epic-04-document-management.md#STORY-04-01]

To avoid server-side PDF processing overhead, convert the PDF's first page to a PNG in the browser using `pdfjs-dist`:

```typescript
// src/lib/pdfToImage.ts
import * as pdfjsLib from "pdfjs-dist";
pdfjsLib.GlobalWorkerOptions.workerSrc = "/pdf.worker.min.mjs"; // serve from public/

export async function pdfFirstPageToBlob(file: File): Promise<Blob> {
  const arrayBuffer = await file.arrayBuffer();
  const pdf = await pdfjsLib.getDocument({ data: arrayBuffer }).promise;
  const page = await pdf.getPage(1);
  const viewport = page.getViewport({ scale: 2.0 }); // 2x for better OCR quality
  const canvas = document.createElement("canvas");
  canvas.width = viewport.width;
  canvas.height = viewport.height;
  const context = canvas.getContext("2d")!;
  await page.render({ canvasContext: context, viewport }).promise;
  return new Promise((resolve) => canvas.toBlob((blob) => resolve(blob!), "image/png"));
}
```

Install: `pnpm add pdfjs-dist`. Serve `pdf.worker.min.mjs` from `public/` or configure via Next.js `webpack` config to copy it.

Decision point noted in AC 10: this approach is recommended to keep server lightweight. An alternative (server-side `sharp` for PDF rasterisation) would require a lambda with native binaries — avoid for Vercel serverless.

### POST /api/vehicles/[id]/documents/parse

[Source: architecture.md#5-api-specification]

Create `src/app/api/vehicles/[id]/documents/parse/route.ts`:

```typescript
export async function POST(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    // Verify vehicle access
    await vehicleService.getWithAccessCheck(params.id, session.user.id);

    // Parse multipart form data
    const formData = await req.formData();
    const file = formData.get("file") as File;
    const docType = formData.get("type") as string;

    if (!file) return NextResponse.json({ error: "No file provided" }, { status: 400 });

    // Validate file size (10MB)
    if (file.size > 10 * 1024 * 1024) {
      return NextResponse.json({ error: "File too large (max 10MB)" }, { status: 400 });
    }

    // Upload to R2 as temp file
    const tempKey = `${session.user.id}/documents/temp/${crypto.randomUUID()}.jpg`;
    const fileBuffer = await file.arrayBuffer();
    await storageService.uploadFile(tempKey, Buffer.from(fileBuffer), file.type);

    // Call Claude API
    const result = await parseDocument(fileBuffer, file.type);

    return NextResponse.json({
      extractedExpiryDate: result.expiryDate ?? null,
      documentType: docType,
      confidence: result.confidence,
      parseStatus: result.expiryDate ? "parsed" : "failed",
      tempR2Key: tempKey,
    });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### parseDocument() in src/lib/anthropic.ts

[Source: architecture.md#7-external-apis (Claude API)]

Add `parseDocument()` to `src/lib/anthropic.ts` (the stub from Story 1.5):

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

export async function parseDocument(
  fileBuffer: ArrayBuffer,
  mimeType: string
): Promise<{ expiryDate: string | null; confidence: "high" | "medium" | "low" | "none" }> {
  const base64 = Buffer.from(fileBuffer).toString("base64");
  const imageMediaType = mimeType.includes("png") ? "image/png" : "image/jpeg";

  const response = await client.messages.create({
    model: "claude-3-haiku-20240307",
    max_tokens: 256,
    messages: [{
      role: "user",
      content: [
        {
          type: "image",
          source: { type: "base64", media_type: imageMediaType, data: base64 },
        },
        {
          type: "text",
          text: `This is an Indian vehicle document. Extract ONLY the expiry date or validity date.
Return JSON: { "expiryDate": "YYYY-MM-DD" or null, "confidence": "high" | "medium" | "low" | "none" }
If no expiry date is found, return null for expiryDate.
Return ONLY valid JSON — no markdown, no explanation.`,
        },
      ],
    }],
  });

  const text = (response.content[0] as { text: string }).text.trim();
  try {
    return JSON.parse(text);
  } catch {
    return { expiryDate: null, confidence: "none" };
  }
}
```

Model: `claude-3-haiku-20240307` (fast + cheap for vision tasks at ~$0.003/parse).

### documentService.create()

[Source: architecture.md#6-components]

Add to `src/services/documentService.ts`:

```typescript
async create(vehicleId: string, userId: string, data: CreateDocumentInput): Promise<Document> {
  let storageUrl: string | null = null;
  let storageKey: string | null = null;

  // Determine storage behavior based on user preference
  const user = await db.query.users.findFirst({ where: eq(users.id, userId) });
  const pref = user?.documentStoragePreference ?? "parse_only";

  if (data.tempR2Key) {
    if (pref === "full_storage") {
      // Move temp file to permanent key
      const permanentKey = `${userId}/documents/${vehicleId}/${crypto.randomUUID()}.jpg`;
      await storageService.copyFile(data.tempR2Key, permanentKey);
      await storageService.deleteFile(data.tempR2Key);
      storageKey = permanentKey;
      storageUrl = storageService.getPublicUrl(permanentKey); // or null if not public
    } else {
      // parse_only: delete temp file
      await storageService.deleteFile(data.tempR2Key).catch((e) =>
        logger.error({ error: e }, "Failed to delete temp R2 file after parse")
      );
    }
  }

  const [doc] = await db.insert(documents).values({
    vehicleId,
    userId,
    type: data.type,
    label: data.label,
    expiryDate: data.expiryDate ? new Date(data.expiryDate) : null,
    storageUrl,
    storageKey,
    parseStatus: data.parseStatus,
    status: computeDocumentStatus(data.expiryDate, user?.notificationWindowDays ?? 30),
  }).returning();

  return doc;
}
```

`computeDocumentStatus()` is a pure utility function — implement in `src/services/documentService.ts` or `src/utils/documentStatus.ts`:

```typescript
function computeDocumentStatus(
  expiryDate: string | null | undefined,
  notificationWindowDays: number
): DocumentStatus {
  if (!expiryDate) return "incomplete";
  const today = new Date();
  const expiry = new Date(expiryDate);
  const daysUntilExpiry = Math.floor((expiry.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));
  if (daysUntilExpiry < 0) return "expired";
  if (daysUntilExpiry <= notificationWindowDays) return "expiring";
  return "valid";
}
```
