# Story 11.2 — Community Moderation (Reported Posts)

**Epic:** 11 — Admin Panel
**Story:** 11.2
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 11.1 (admin auth), Story 9.5 (reported posts in DB)

## Story

**As an** admin,
**I want** to review reported and auto-hidden posts and take moderation actions,
**so that** the community remains safe and on-topic.

---

## Acceptance Criteria

1. - [ ] Admin page: `src/app/admin/reported/page.tsx`
2. - [ ] Moderation queue shows: auto-hidden posts (`is_hidden=true`) + high-report-count posts (>= 3 reports for MVP display threshold, separate from auto-hide threshold)
3. - [ ] Each entry shows: post title, report count, reasons breakdown (e.g. "Spam ×3, Harassment ×1"), post content preview (first 200 chars of description)
4. - [ ] Actions per post (with ConfirmModal where noted):
   - **Restore**: set `is_hidden=false` — no confirmation needed
   - **Remove post**: permanent DELETE — ConfirmModal
   - **Warn user**: create warning record + in-app + email notification to post author — ConfirmModal
   - **Suspend user**: input suspension duration (days); set `status='suspended'`, `suspended_until=now()+days`; in-app + email
   - **Ban user**: set `status='banned'`; permanent; in-app + email — ConfirmModal
5. - [ ] `GET /api/admin/posts/reported` — returns moderation queue
6. - [ ] `PATCH /api/admin/posts/[id]` — restore or remove
7. - [ ] `PATCH /api/admin/users/[id]` — warn, suspend, ban
8. - [ ] `adminService.restorePost(postId)`, `adminService.removePost(postId)`, `adminService.warnUser(userId)`, `adminService.suspendUser(userId, days)`, `adminService.banUser(userId)` in `src/services/adminService.ts`
9. - [ ] All actions logged: `logger.info({ action, targetId, adminId }, "admin action")` (MVP — no separate log table)
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

### adminService.getReportedPosts()

[Source: architecture.md#6-components]

Add to `src/services/adminService.ts`:

```typescript
async getReportedPosts(): Promise<ReportedPost[]> {
  // Posts that are either hidden OR have >= 3 reports
  const postsWithReports = await db.query.posts.findMany({
    with: {
      user: { columns: { id: true, name: true, email: true } },
      reports: true,
    },
  });

  return postsWithReports
    .filter(p => p.isHidden || p.reports.length >= 3)
    .map(p => {
      const reasonCounts = p.reports.reduce((acc, r) => {
        acc[r.reason] = (acc[r.reason] ?? 0) + 1;
        return acc;
      }, {} as Record<string, number>);

      return {
        id: p.id,
        title: p.title,
        descriptionPreview: p.description.slice(0, 200),
        reportCount: p.reports.length,
        reasonBreakdown: reasonCounts,
        isHidden: p.isHidden,
        author: p.user,
      };
    })
    .sort((a, b) => b.reportCount - a.reportCount);
}
```

### Admin Action Methods

[Source: architecture.md#6-components]

Add to `src/services/adminService.ts`:

```typescript
async restorePost(postId: string, adminId: string): Promise<void> {
  await db.update(posts).set({ isHidden: false }).where(eq(posts.id, postId));
  logger.info({ action: "restore_post", postId, adminId }, "admin action");
}

async removePost(postId: string, adminId: string): Promise<void> {
  const post = await db.query.posts.findFirst({ where: eq(posts.id, postId) });
  if (!post) throw new NotFoundError("Post not found");
  // Delete R2 images
  if (post.imageUrls?.length) {
    await Promise.allSettled(post.imageUrls.map(key => deleteObject(key)));
  }
  await db.delete(posts).where(eq(posts.id, postId));
  logger.info({ action: "remove_post", postId, adminId }, "admin action");
}

async warnUser(userId: string, adminId: string): Promise<void> {
  await db.update(users).set({ status: "warned" }).where(eq(users.id, userId));
  await notificationService.create(
    userId, "admin_warning",
    "Account warning",
    "Your account has received a warning for violating community guidelines.",
    "/community"
  );
  const user = await userService.getById(userId);
  await sendEmail(user.email, "Account warning — MotoYaar",
    "<p>Your account has received a warning for violating community guidelines.</p>");
  logger.info({ action: "warn_user", userId, adminId }, "admin action");
}

async suspendUser(userId: string, days: number, adminId: string): Promise<void> {
  const suspendedUntil = new Date(Date.now() + days * 86_400_000);
  await db.update(users)
    .set({ status: "suspended", suspendedUntil })
    .where(eq(users.id, userId));
  await notificationService.create(userId, "admin_suspension",
    "Account suspended",
    `Your account has been suspended for ${days} day${days === 1 ? "" : "s"}.`,
    "/community");
  const user = await userService.getById(userId);
  await sendEmail(user.email, "Account suspended — MotoYaar",
    `<p>Your account has been suspended for ${days} days for violating community guidelines.</p>`);
  logger.info({ action: "suspend_user", userId, days, adminId }, "admin action");
}

async banUser(userId: string, adminId: string): Promise<void> {
  await db.update(users).set({ status: "banned" }).where(eq(users.id, userId));
  await notificationService.create(userId, "admin_ban",
    "Account banned",
    "Your account has been permanently banned for violating community guidelines.",
    "/community");
  logger.info({ action: "ban_user", userId, adminId }, "admin action");
}
```

### Moderation Queue UI

[Source: front-end-spec.md#3-screen-by-screen-specifications]

Desktop-first layout (admin is desktop-only):

```tsx
// src/app/admin/reported/page.tsx
export default async function ReportedPostsPage() {
  const session = await getAdminSession();
  if (!session.admin) notFound();

  const posts = await adminService.getReportedPosts();

  return (
    <div>
      <h1 className="text-2xl font-bold mb-6">Moderation Queue ({posts.length})</h1>
      <div className="space-y-4">
        {posts.map(post => (
          <ReportedPostCard key={post.id} post={post} adminId={session.admin!.id} />
        ))}
      </div>
    </div>
  );
}
```

`ReportedPostCard` is a `"use client"` component that handles action buttons and confirmation modals.
