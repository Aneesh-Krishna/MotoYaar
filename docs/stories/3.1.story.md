# Story 3.1 — Bottom Navigation (Mobile)

**Epic:** 03 — Vehicle Management
**Story:** 3.1
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 2.1 (Auth / NextAuth session)

---

## Story

**As a** mobile user,
**I want** a persistent bottom navigation bar with five tabs,
**so that** I can switch between the app's main sections with a single tap from anywhere in the app.

---

## Acceptance Criteria

1. - [ ] Bottom nav is visible on all authenticated pages inside the `(app)` route group
2. - [ ] Five tabs in order: Home (Dashboard), Garage, Community, Trips, Profile — each with icon and label
3. - [ ] Active tab is highlighted: icon and label rendered in primary orange `#F97316`; inactive tabs show icon and label in muted gray `#6B7280`
4. - [ ] Bottom nav height is 64px with `pb-safe` (iOS home indicator clearance)
5. - [ ] At `lg:` breakpoint (1024px+), bottom nav is replaced by a fixed 240px left sidebar nav; bottom nav is hidden
6. - [ ] Bell icon in top bar (TopBar component); tapping it opens a NotificationDrawer stub (empty drawer — "No notifications yet")
7. - [ ] FAB (Floating Action Button) is context-sensitive: Garage tab → "Add Vehicle" (links to `/garage/new`), Community tab → "New Post" (stub), Trips tab → "Add Trip" (links to `/trips/new` stub), Home/Profile → FAB hidden
8. - [ ] ARIA: nav element has `role="tablist"`; each tab button has `role="tab"` and `aria-selected="true|false"`
9. - [ ] Tab labels use Lucide icons at 24px — outline style when inactive, filled/colored when active
10. - [ ] `pnpm typecheck` and `pnpm lint` pass after changes

### Tab Configuration

[Source: front-end-spec.md#2-information-architecture--navigation]

```typescript
// Centralise tab config — import this in both BottomNav and SidebarNav
export const NAV_TABS = [
  { href: "/",          label: "Home",      icon: "Home"    },
  { href: "/garage",    label: "Garage",    icon: "Car"     },
  { href: "/community", label: "Community", icon: "Users"   },
  { href: "/trips",     label: "Trips",     icon: "Map"     },
  { href: "/profile",   label: "Profile",   icon: "User"    },
] as const;

export const FAB_CONFIG = {
  "/garage":    { label: "Add Vehicle", href: "/garage/new"   },
  "/community": { label: "New Post",    href: "/community/new" },
  "/trips":     { label: "Add Trip",    href: "/trips/new"    },
} as const;
```

Use `usePathname()` from `next/navigation` to determine the active tab. Match on `pathname === href` for exact match on `/`; use `pathname.startsWith(href)` for all others.

### BottomNav Component

[Source: front-end-spec.md#3-visual-design--branding]

`src/components/layout/BottomNav.tsx` — mark `"use client"` (uses `usePathname`).

Key implementation details:
- Fixed to bottom of viewport: `fixed bottom-0 left-0 right-0 z-50`
- Height: `h-16` (64px) with `pb-safe` via Tailwind plugin or CSS env padding
- Background: `bg-white border-t border-gray-200`
- On `lg:` screens: `lg:hidden` — sidebar replaces it
- Tab icons from Lucide React at `size={24}`. Use dynamic import pattern or import each icon by name.
- Active state: text and icon in `text-orange-500`; inactive: `text-gray-500`
- Each tab `<button>` navigates using `useRouter().push(href)` or wraps a Next.js `<Link>`

ARIA:
```tsx
<nav role="tablist" aria-label="Main navigation">
  {NAV_TABS.map(tab => (
    <Link
      key={tab.href}
      href={tab.href}
      role="tab"
      aria-selected={isActive(tab.href)}
      aria-label={tab.label}
    >
      ...
    </Link>
  ))}
</nav>
```

### FAB Component

[Source: front-end-spec.md#2-information-architecture--navigation]

FAB is a floating button rendered inside `(app)/layout.tsx`, positioned above the bottom nav: `fixed bottom-20 right-4 z-50`.

```tsx
// Determine FAB config based on current route
const fabEntry = Object.entries(FAB_CONFIG).find(([key]) =>
  key === "/" ? pathname === key : pathname.startsWith(key)
);
if (!fabEntry) return null; // Home and Profile — FAB hidden

return (
  <Link href={fabEntry[1].href}>
    <button
      className="bg-orange-500 text-white rounded-full shadow-lg w-14 h-14 flex items-center justify-center"
      aria-label={fabEntry[1].label}
    >
      <Plus size={24} />
    </button>
  </Link>
);
```

### Layout Wiring

[Source: architecture.md#12-unified-project-structure]

In `src/app/(app)/layout.tsx`, the structure should be:

```tsx
export default function AppLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="min-h-screen bg-[#FAFAF8]">
      <TopBar />
      <SidebarNav />
      <main className="pt-14 pb-20 lg:pl-60 lg:pt-0 lg:pb-0">
        {children}
      </main>
      <BottomNav />
      <FAB />  {/* or inline FAB logic */}
    </div>
  );
}
```

`pt-14` accounts for TopBar height on mobile. `pb-20` accounts for BottomNav height + safe area. `lg:pl-60` shifts content right of sidebar on desktop.

### Accessibility

[Source: front-end-spec.md#7-accessibility--performance]

- Never rely on color alone to convey active state — use both color and visible label weight change
- All icon-only interactive elements must have `aria-label`
- Focus ring must be visible on keyboard navigation: use Tailwind `focus-visible:ring-2 focus-visible:ring-orange-500`
- Minimum tap target: 44×44px (bottom nav tabs should fill their allocated space)
