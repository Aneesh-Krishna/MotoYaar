## Story

**As a** rider on a route,
**I want** to see live traffic, find nearby fuel stations and rest stops, and have map markers clustered when they overlap,
**so that** I can make informed decisions about my route and stops without leaving the app.

---

1. - [ ] A **traffic layer toggle** is available on the map; default ON during active navigation, OFF when browsing
2. - [ ] Traffic data is sourced from Google Maps Platform real-time traffic
3. - [ ] ETA during navigation uses `drivingOptions.trafficModel = 'bestguess'` (traffic-aware ETA)

10. - [ ] When multiple markers (group members, POIs) are close together, they are **clustered** using `@googlemaps/markerclusterer` with a count badge
11. - [ ] Tapping a cluster expands it to show individual markers
12. - [ ] Custom map style suppresses irrelevant POIs (shopping, restaurants in general) while emphasising fuel stations, highways, and rest areas
13. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### вљ пёЏ FINANCIAL RISK вЂ” NEARBY SEARCH CACHE IS NON-NEGOTIABLE

The Nearby Search API costs $32 per 1,000 requests вЂ” 4вЂ“6x more expensive than every other Google Maps API. A bug that bypasses the 1-hour cache (e.g., a React component re-mounting) can generate hundreds of dollars of charges in a single day.

**Hard requirements (from PRD Section 7.1.5):**
1. Cache must be verified end-to-end **before** this story deploys to any environment with real API keys.
2. A dedicated integration test must assert that repeated calls within 1 hour return cached results.
3. The feature must be behind a feature flag until caching is confirmed.
4. Budget alert at в‚№2,000/month (warn) + в‚№5,000/month (hard cutoff) вЂ” confirmed in Story 17.1.

### Nearby Places Cache Design

Cache key: `nearby:{lat_rounded_3dp}:{lng_rounded_3dp}:{type}` (rounding lat/lng to 3 decimal places ~111m grid means nearby triggers from approximately the same spot reuse cache).

Store in IndexedDB (via `idb`) вЂ” NOT in `map_cache` Postgres table (avoids DB writes for per-session data).

```typescript
// src/lib/maps/nearbyCache.ts
interface NearbyCacheEntry {
  results: google.maps.places.PlaceResult[];
  fetchedAt: number; // Date.now()
}

const CACHE_TTL_MS = 60 * 60 * 1000; // 1 hour

export async function getNearby(lat: number, lng: number, type: string) {
  const key = `nearby:${lat.toFixed(3)}:${lng.toFixed(3)}:${type}`;
  const db = await openNearbyDb();
  const cached = await db.get('nearby', key);
  if (cached && Date.now() - cached.fetchedAt < CACHE_TTL_MS) {
    return cached.results; // cache hit вЂ” NO API call
  }
  const results = await fetchNearbyFromApi(lat, lng, type); // API call
  await db.put('nearby', { results, fetchedAt: Date.now() }, key);
  return results;
}
```

### Nearby Places API Call (Server-Side Proxy)

Nearby Search must be proxied through a server route to keep the API key server-side:

```
POST /api/maps/nearby   вЂ” body: { lat, lng, type, radius }
                         вЂ” requires auth (JWT)
                         вЂ” calls Google Places Nearby Search API
                         вЂ” NOT cached server-side (cache is client IndexedDB)
```

### Custom Map Style

A custom style JSON is applied to suppress irrelevant POIs and emphasise riding-relevant features. Store in `src/lib/maps/mapStyles.ts`. Apply to the `GoogleMap` component via `options.styles`.

Key style rules:
- Hide: `poi.attraction`, `poi.business`, `poi.medical`, `poi.place_of_worship`, `poi.school`
- Show: `poi.park` (rest areas), road labels, highway markers
- Emphasise: fuel station icons (keep visible at zoom в‰Ґ 13)
