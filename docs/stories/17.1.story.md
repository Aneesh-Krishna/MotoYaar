# Story 17.1 — Map Embed, Place Search & Directions

**Epic:** 17 — Maps, Group Rides & Offline Navigation
**Story:** 17.1
**Status:** Done
**Created:** 2026-05-17
**Depends On:** Story 1.1 (Foundation — Supabase client, Next.js scaffold), Story 2.1 (Auth — session)

## Story

**As a** rider planning a trip,
**I want** to open an embedded map inside MotoYaar with place search and turn-by-turn directions,
**so that** I can navigate without switching to a separate maps app and losing my trip context.

---

## Acceptance Criteria

1. - [ ] A **Maps** tab or page is accessible from the main app navigation
2. - [ ] The map renders using `@react-google-maps/api` centred on the user's current GPS location on load
3. - [ ] Map supports zoom levels 5–20, pinch-to-zoom, and pan on mobile
4. - [ ] Map type toggles are available: Roadmap (default), Satellite, Terrain, Hybrid
5. - [ ] Dark mode map style activates automatically when system dark mode is on, and can be toggled manually
6. - [ ] A search bar provides **Google Places Autocomplete** with session token management — a single Google API charge per completed search, not per keystroke (hard requirement)
7. - [ ] Search results are scoped to India by default
8. - [ ] Recent searches (up to 20) are stored in IndexedDB and shown below the search bar before user types
9. - [ ] Selecting a search result places a destination marker and triggers A→B routing via Google Directions API using current location as origin
10. - [ ] ETA is displayed prominently (large badge) during active navigation
11. - [ ] A turn-by-turn instruction list is available in an expandable drawer
12. - [ ] Current instruction is shown as a HUD overlay at the top of the map during navigation
13. - [ ] Distance to next turn updates as the rider moves
14. - [ ] When the rider deviates >150m from route, new directions are automatically requested (online re-routing)
15. - [ ] Speed in km/h (from GPS, not speedometer) is displayed during navigation
16. - [ ] Online voice navigation uses `SpeechSynthesis` API with locale `en-IN`, mutable via UI toggle
17. - [ ] Maps JS SDK is lazy-loaded — only loaded on routes that render a map (not on every page)
18. - [ ] A Google Cloud budget alert is confirmed set at **₹2,000/month (warn)** and **₹5,000/month (hard cutoff)** before this story ships to any environment with real API keys
19. - [ ] `pnpm typecheck` and `pnpm lint` pass
20. - [ ] **Also:** Supabase Realtime Broadcast proof-of-concept is demonstrated working (can send/receive a broadcast message on a test channel) — required to de-risk Story 17.3

### Session Token Pattern for Autocomplete (Non-Negotiable)

Google charges per-session for Autocomplete, not per-keystroke, **only when session tokens are used**. Without them, every keystroke is a billable request.

```typescript
// Create once per search session (new token when user clears/starts new search)
const sessionToken = new google.maps.places.AutocompleteSessionToken();

// Pass on every autocomplete request
const autocompleteService = new google.maps.places.AutocompleteService();
autocompleteService.getPlacePredictions(
  { input, sessionToken, componentRestrictions: { country: 'in' } },
  callback
);

// When user selects a result → call getDetails with the SAME token
// After getDetails call, the session ends and token must be refreshed
placesService.getDetails({ placeId, sessionToken }, callback);
```

[Source: PRD Section 7.1.2]

### API Response Caching (Non-Negotiable)

Directions API responses must be cached by route hash in `map_cache` table:
- Traffic-sensitive routes: 30-minute TTL
- Static (no traffic model): 24-hour TTL

```sql
-- map_cache table (already in PRD Section 10 schema)
cache_key TEXT PK,         -- SHA256 hash of origin+destination+travelMode
cache_type TEXT,           -- 'directions' | 'geocode' | 'autocomplete'
response_data JSONB,
created_at TIMESTAMPTZ,
expires_at TIMESTAMPTZ,
hit_count INTEGER DEFAULT 0
```

Geocoding responses: 7-day TTL.

[Source: PRD Section 7.1.7, PRD Section 10]

### DB Schema — New Tables This Story

```sql
-- API response cache
CREATE TABLE map_cache (
  cache_key     TEXT PRIMARY KEY,
  cache_type    TEXT NOT NULL CHECK (cache_type IN ('autocomplete', 'directions', 'geocode')),
  response_data JSONB NOT NULL,
  created_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  expires_at    TIMESTAMPTZ NOT NULL,
  hit_count     INTEGER NOT NULL DEFAULT 0
);
CREATE INDEX idx_map_cache_expires ON map_cache(expires_at);

-- User saved/bookmarked places
CREATE TABLE saved_places (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id    UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  name       TEXT NOT NULL,
  address    TEXT,
  lat        DOUBLE PRECISION NOT NULL,
  lng        DOUBLE PRECISION NOT NULL,
  place_id   TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_saved_places_user_id ON saved_places(user_id);
```
