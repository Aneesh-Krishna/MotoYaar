# Story 5.4 — Delete Expense

**Epic:** 05 — Expense Tracking
**Story:** 5.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 5.3 (Edit Expense — delete is accessible from the edit view)

---

## Story

**As a** user,
**I want** to delete an expense I logged in error,
**so that** my spend records stay accurate.

---

## Acceptance Criteria

1. - [ ] Delete is accessible from the expense edit view (trash icon in the BottomSheet header or a "Delete Expense" button at the bottom of the edit form)
2. - [ ] `ConfirmModal` shown: "Delete this expense? This cannot be undone."
3. - [ ] Trip-linked expenses cannot be deleted independently: if `expense.tripId` is not null, show blocking message "This expense belongs to a trip. Delete the trip to remove it." — no delete action available
4. - [ ] `DELETE /api/expenses/[id]` called on confirm
5. - [ ] `expenseService.delete(expenseId, userId)` — ownership check + trip-linkage block
6. - [ ] If `expense.receiptUrl` exists in R2: delete the receipt file from R2 before deleting the DB record (Story 5.5 implements receipt storage — stub this call here)
7. - [ ] On confirm: expense deleted; BottomSheet closed; expense list refreshes via `router.refresh()`
8. - [ ] `pnpm typecheck` and `pnpm lint` pass

### Trip-Linked Block

[Source: epic-05-expense-tracking.md#STORY-05-04]

The trip-linked check happens at both the UI level and the service level:

**UI level:** Delete button not rendered when `expense.tripId !== null` (Story 5.3 read-only indicator covers this case).

**Service level:** `expenseService.delete()` also checks and throws a descriptive error:

```typescript
if (expense.tripId) {
  throw new ForbiddenError("Trip-linked expenses can only be deleted by deleting the trip");
}
```

This double-check prevents API-level abuse.

### expenseService.delete()

[Source: architecture.md#6-components]

Add to `src/services/expenseService.ts`:

```typescript
async delete(expenseId: string, userId: string): Promise<void> {
  const expense = await db.query.expenses.findFirst({
    where: eq(expenses.id, expenseId),
  });

  if (!expense) throw new NotFoundError("Expense not found");
  if (expense.userId !== userId) throw new ForbiddenError("You do not own this expense");

  // Block delete for trip-linked expenses
  if (expense.tripId) {
    throw new ForbiddenError("Trip-linked expenses can only be deleted by deleting the trip");
  }

  // Delete receipt from R2 if it exists
  // (receiptKey stored on expense — Story 5.5 adds this column)
  // For now: stub this block with a comment
  if (expense.receiptUrl) {
    // TODO: Story 5.5 — delete R2 receipt file using expense.receiptKey
    // await storageService.deleteFile(expense.receiptKey);
    logger.info({ expenseId }, "Receipt R2 deletion: stub — implement in Story 5.5");
  }

  // Delete expense record
  await db.delete(expenses).where(eq(expenses.id, expenseId));
}
```

The R2 receipt deletion is stubbed here because the `receiptKey` column on `expenses` is added in Story 5.5. The stub ensures the code structure is in place so Story 5.5 only needs to fill in the `storageService.deleteFile()` call.

### ConfirmModal Usage

[Source: architecture.md#6-components]

Reuse `ConfirmModal` (created in Story 3.6):

```tsx
<ConfirmModal
  open={showDeleteModal}
  onClose={() => setShowDeleteModal(false)}
  onConfirm={handleDelete}
  title="Delete this expense?"
  description="This cannot be undone."
  confirmLabel="Delete Expense"
  isDestructive={true}
  isLoading={isDeleting}
/>
```

Delete handler in `ExpenseForm` (edit mode):

```typescript
const handleDelete = async () => {
  setIsDeleting(true);
  try {
    await deleteExpense(expense!.id);
    toast.success("Expense deleted");
    onSaved(); // closes BottomSheet + triggers router.refresh()
  } catch (error) {
    if (error instanceof ApiError && error.status === 403) {
      toast.error("This expense is linked to a trip. Delete the trip to remove it.");
    } else {
      toast.error("Failed to delete expense. Try again.");
    }
  } finally {
    setIsDeleting(false);
    setShowDeleteModal(false);
  }
};
```
