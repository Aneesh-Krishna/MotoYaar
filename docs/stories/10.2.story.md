# Story 10.2 — Accept Invite (Existing User)

**Epic:** 10 — Vehicle Sharing
**Story:** 10.2
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 10.1 (invite records and vehicle_access table exist)

## Story

**As an** existing MotoYaar user,
**I want** to accept a vehicle invite from a link in my email,
**so that** the shared vehicle appears in my garage for view-only access.

---

## Acceptance Criteria

1. - [ ] Route `src/app/invites/[token]/page.tsx` — outside the `(app)` group (publicly accessible)
2. - [ ] If logged in and token valid: resolve invite → create `VehicleAccess` record → redirect to vehicle detail page
3. - [ ] If token expired (`expires_at < now()`): show "This invite has expired. Ask the vehicle owner to send a new one."
4. - [ ] If token already accepted (`status='accepted'`): show "This invite has already been accepted."
5. - [ ] If token not found: show generic "Invalid invite link."
6. - [ ] Accepted vehicle appears in invitee's Garage tab under "Shared with me" section
7. - [ ] View-only mode: all vehicle detail tabs visible; no add/edit/delete actions (`isOwner=false`)
8. - [ ] `POST /api/invites/[token]/accept` (or `GET` redirect pattern) — processes the invite acceptance
9. - [ ] `vehicleInviteService.acceptInvite(token, userId)` — validates token + creates `VehicleAccess`
10. - [ ] Garage list fetches owned vehicles + vehicles from `VehicleAccess` records for user
11. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### Invite Acceptance Route

[Source: architecture.md#12-unified-project-structure]

Create `src/app/invites/[token]/page.tsx` in the root (NOT inside `(app)`):

```tsx
// src/app/invites/[token]/page.tsx
import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";
import { vehicleInviteService } from "@/services/vehicleInviteService";
import { redirect } from "next/navigation";

export default async function InvitePage({ params }: { params: { token: string } }) {
  const session = await getServerSession(authOptions);

  if (!session) {
    // Not logged in — redirect to login with invite token in URL
    redirect(`/login?inviteToken=${params.token}`);
  }

  // Logged in — try to accept
  let result;
  try {
    result = await vehicleInviteService.acceptInvite(params.token, session.user.id);
  } catch (err) {
    return <InviteErrorView error={err} />;
  }

  redirect(`/garage/${result.vehicleId}`);
}
```

`InviteErrorView` renders the appropriate error message based on the error type.

### vehicleInviteService.acceptInvite()

[Source: architecture.md#6-components]

Add to `src/services/vehicleInviteService.ts`:

```typescript
async acceptInvite(
  token: string,
  userId: string
): Promise<{ vehicleId: string }> {
  const invite = await db.query.vehicleInvites.findFirst({
    where: eq(vehicleInvites.token, token),
  });

  if (!invite) throw new NotFoundError("Invite not found");

  if (invite.status === "accepted") {
    throw new ConflictError("This invite has already been accepted.");
  }

  if (new Date(invite.expiresAt) < new Date()) {
    throw new ConflictError("This invite has expired.");
  }

  // Create VehicleAccess (upsert — user may have had access before)
  await db.insert(vehicleAccess)
    .values({
      vehicleId: invite.vehicleId,
      userId,
      accessLevel: "view",
    })
    .onConflictDoNothing(); // UNIQUE(vehicle_id, user_id)

  // Mark invite as accepted
  await db.update(vehicleInvites)
    .set({ status: "accepted" })
    .where(eq(vehicleInvites.id, invite.id));

  return { vehicleId: invite.vehicleId };
}
```

### Error View Component

```tsx
interface InviteErrorViewProps {
  error: unknown;
}

export function InviteErrorView({ error }: InviteErrorViewProps) {
  let message = "Invalid invite link.";
  if (error instanceof ConflictError) {
    message = error.message; // "already accepted" or "expired"
  } else if (error instanceof NotFoundError) {
    message = "Invalid invite link. This invite does not exist.";
  }

  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50 px-4">
      <div className="max-w-sm w-full bg-white rounded-2xl p-8 text-center shadow-sm">
        <div className="w-12 h-12 bg-red-100 rounded-full flex items-center justify-center mx-auto mb-4">
          <XCircle className="text-red-500" size={24} />
        </div>
        <h1 className="text-lg font-bold text-gray-900 mb-2">Invite issue</h1>
        <p className="text-sm text-gray-500 mb-6">{message}</p>
        <Link href="/garage">
          <Button variant="outline" className="w-full">Go to Garage</Button>
        </Link>
      </div>
    </div>
  );
}
```

### Garage — "Shared with me" Section

[Source: architecture.md#6-components]

Extend `vehicleService.listByUser(userId)` to also fetch vehicles shared via `vehicle_access`:

```typescript
async listByUser(userId: string): Promise<{ owned: Vehicle[]; shared: Vehicle[] }> {
  const [owned, sharedAccess] = await Promise.all([
    db.query.vehicles.findMany({
      where: eq(vehicles.userId, userId),
    }),
    db.query.vehicleAccess.findMany({
      where: eq(vehicleAccess.userId, userId),
      with: { vehicle: true },
    }),
  ]);

  return {
    owned,
    shared: sharedAccess.map(a => a.vehicle),
  };
}
```

In the Garage page, render two sections:
1. "My Vehicles" — owned vehicles
2. "Shared with me" — only shown if `shared.length > 0`

### View-Only Mode

[Source: architecture.md#6-components]

When a user accesses a vehicle they don't own (via `VehicleAccess`), `getWithAccessCheck(vehicleId, userId)` should return the vehicle with an `isOwner: false` flag:

```typescript
async getWithAccessCheck(vehicleId: string, userId: string): Promise<VehicleWithAccess> {
  const vehicle = await db.query.vehicles.findFirst({
    where: eq(vehicles.id, vehicleId),
  });
  if (!vehicle) throw new NotFoundError("Vehicle not found");

  if (vehicle.userId === userId) return { ...vehicle, isOwner: true };

  // Check shared access
  const access = await db.query.vehicleAccess.findFirst({
    where: and(eq(vehicleAccess.vehicleId, vehicleId), eq(vehicleAccess.userId, userId)),
  });
  if (!access) throw new ForbiddenError("Access denied");

  return { ...vehicle, isOwner: false };
}
```

Vehicle Detail page passes `isOwner` to child components to hide edit/delete/add buttons when `false`.
