# Story 12.3 — Currency & Display Settings

**Epic:** 12 — Profile & Settings
**Story:** 12.3
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 12.1 (settings page exists)

## Story

**As a** user,
**I want** to set my preferred currency,
**so that** all expense amounts and report totals are shown in the currency I use.

---

### CurrencySetting Component

[Source: front-end-spec.md#5-component-library--ui-patterns]

```tsx
const CURRENCIES = [
  { code: "INR", label: "Indian Rupee", symbol: "₹" },
  { code: "USD", label: "US Dollar", symbol: "$" },
  { code: "EUR", label: "Euro", symbol: "€" },
  { code: "GBP", label: "British Pound", symbol: "£" },
  { code: "AED", label: "UAE Dirham", symbol: "د.إ" },
  { code: "SGD", label: "Singapore Dollar", symbol: "S$" },
];

function CurrencySetting({
  currentCurrency,
  onSave,
}: {
  currentCurrency: string;
  onSave: () => void;
}) {
  const [saving, setSaving] = useState(false);

  const handleChange = async (currency: string) => {
    setSaving(true);
    await fetch("/api/users/me", {
      method: "PATCH",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ currency }),
    });
    await onSave(); // refresh session
    setSaving(false);
    toast.success("Currency updated");
  };

  return (
    <div className="flex items-center justify-between px-4 py-4">
      <div>
        <p className="text-sm font-medium text-gray-900">Currency</p>
        <p className="text-xs text-gray-500">Used for expense amounts and reports</p>
      </div>
      <Select value={currentCurrency} onValueChange={handleChange} disabled={saving}>
        <SelectTrigger className="w-40">
          <SelectValue />
        </SelectTrigger>
        <SelectContent>
          {CURRENCIES.map(c => (
            <SelectItem key={c.code} value={c.code}>
              {c.symbol} {c.label}
            </SelectItem>
          ))}
        </SelectContent>
      </Select>
    </div>
  );
}
```

`onSave()` calls `update()` from `useSession()` to refresh the session JWT with the new currency value. The session type must include `currency` on `user` — extend the NextAuth types in `src/types/next-auth.d.ts` if needed.

### Session Type Extension

[Source: architecture.md#2-tech-stack]

Extend NextAuth types to include `currency`:

```typescript
// src/types/next-auth.d.ts (create or update)
import NextAuth from "next-auth";

declare module "next-auth" {
  interface Session {
    user: {
      id: string;
      currency: string;
      onboardingCompleted: boolean;
      // ... other custom fields
    } & DefaultSession["user"];
  }
}
```

In `src/lib/auth.ts`, ensure the session callback includes `currency`:

```typescript
callbacks: {
  session({ session, token }) {
    if (token) {
      session.user.id = token.id as string;
      session.user.currency = token.currency as string ?? "INR";
      // ...
    }
    return session;
  },
  jwt({ token, user }) {
    if (user) {
      token.currency = (user as any).currency ?? "INR";
    }
    return token;
  },
}
```
