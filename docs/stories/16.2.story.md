# Story 16.2 — Log Group Expense with Selective Splitting

**Epic:** 16 — Groups & Club Features
**Story:** 16.2
**Status:** Ready for Review
**Created:** 2026-04-28
**Depends On:** Story 16.1 (Group Expense Session must exist)

---

## Story

**As a** member of a group expense session,
**I want** to log an expense and choose exactly which group members it should be split among,
**so that** individual expenses that only apply to some riders (e.g. juice for two people) are split fairly.

---

## Acceptance Criteria

1. - [ ] Any session member (not just the creator) can tap **"Add Expense"** within a group session
2. - [ ] Expense form fields: **Who paid?** (member picker — defaults to "Me"), **Amount** (required, decimal), **Description** (required, max 100 chars), **Category** (Food | Fuel | Stay | Toll | Misc), **Split between** (multi-select of all session members — all selected by default)
3. - [ ] In **"Split between"**, the user can deselect individual members — only selected members share this expense
4. - [ ] At least one member must be selected in "Split between" (cannot submit with zero members)
5. - [ ] The person who paid (`paid_by`) is **not** automatically included in "Split between" — the payer can be excluded from the split if they chose to cover the cost entirely for others
6. - [ ] The expense is split **equally** among all selected members (equal split only for MVP)
7. - [ ] After saving, the expense appears in the session's item list showing: description, amount, who paid, how many members it was split between, and the per-person share
8. - [ ] Any member can edit an expense they logged; the creator can edit any expense
9. - [ ] Any member can delete an expense they logged; the creator can delete any expense
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

## Dev Notes

### DB Schema

See [growth-features-outcomes.md §4.4](../growth-features-outcomes.md):
```sql
CREATE TABLE group_expense_items (
  id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id           UUID NOT NULL REFERENCES group_expense_sessions(id) ON DELETE CASCADE,
  logged_by            UUID NOT NULL REFERENCES users(id),
  paid_by              UUID NOT NULL REFERENCES users(id),
  amount               NUMERIC(12,2) NOT NULL,
  description          TEXT NOT NULL,
  category             TEXT NOT NULL CHECK (category IN ('Food','Fuel','Stay','Toll','Misc')),
  included_user_ids    UUID[] NOT NULL,   -- members this expense is split among
  created_at           TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at           TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_gei_session_id ON group_expense_items(session_id);
```

### Per-Person Share Calculation

The per-person share is not stored — computed on read:
```typescript
const perPerson = item.amount / item.includedUserIds.length;
```

Equal split only. Unequal split (custom amounts per person) is a post-MVP enhancement.

### Expense Item List Display

```
[Description]                              [Amount]
Paid by: Rahul     Split: 3 people     ₹XX/person
Category: Food
```

For the current user, if they are in `included_user_ids`: highlight their share in the display.

### Authorization

- `logged_by === requestingUser` OR `session.created_by === requestingUser` → can edit/delete
- Any session member → can add a new item
- Non-members → 403 on all item operations

### API Routes

```
GET    /api/group-expenses/[id]/items              — list all items
POST   /api/group-expenses/[id]/items              — add item
PATCH  /api/group-expenses/[id]/items/[itemId]     — edit item
DELETE /api/group-expenses/[id]/items/[itemId]     — delete item
```

### Zod Schema

```typescript
export const createGroupExpenseItemSchema = z.object({
  paidBy: z.string().uuid(),
  amount: z.number().positive(),
  description: z.string().min(1).max(100),
  category: z.enum(['Food', 'Fuel', 'Stay', 'Toll', 'Misc']),
  includedUserIds: z.array(z.string().uuid()).min(1),
});
```
