# Story 2.5 ‚Äî Auth-Gated Route Protection

**Epic:** 02 ‚Äî Authentication & Onboarding
**Story:** 2.5
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 2.1 (Auth/middleware)

## Story

**As a** developer,
**I want** all non-community routes protected by auth middleware,
**so that** unauthenticated users cannot access personal data pages.

---

## Acceptance Criteria

1. - [ ] Next.js middleware protects all routes except `/login`, `/community` (read-only), `/walkthrough` and public static pages
2. - [ ] Unauthenticated access to protected routes redirects to `/login?callbackUrl=[originalUrl]`
3. - [ ] After login, user is redirected back to the originally requested URL (via `callbackUrl`)
4. - [ ] New users with empty username who bypass `/onboarding` are redirected back to `/onboarding`
5. - [ ] Banned users attempting to log in are redirected to a "Your account has been suspended" page

## Dev Notes

### Middleware Configuration

[Source: architecture.md#10-frontend-architecture]

Update `src/middleware.ts` ‚Äî the stub from Story 1.1 had a basic matcher. Now complete it:

```typescript
import { withAuth } from "next-auth/middleware";
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default withAuth(
  function middleware(req) {
    const { token } = req.nextauth;
    const { pathname } = req.nextUrl;

    // New user without username ‚Üí redirect to onboarding
    // (token.username is added to JWT in Story 2.1 authOptions)
    if (token && !token.username && pathname !== "/onboarding" && pathname !== "/walkthrough") {
      return NextResponse.redirect(new URL("/onboarding", req.url));
    }

    return NextResponse.next();
  },
  {
    callbacks: {
      authorized: ({ token }) => !!token,
    },
  }
);

export const config = {
  matcher: [
    /*
     * Protect all routes EXCEPT:
     * - /login (auth page)
     * - /community and /community/* (guest read access)
     * - /api/auth/* (NextAuth routes)
     * - /_next/* (Next.js internals)
     * - /icons/*, /manifest.json (PWA static files)
     */
    "/((?!login|community|api/auth|_next|icons|manifest\\.json|favicon\\.ico).*)",
  ],
};
```

### JWT Token Extension

[Source: architecture.md#nextauth-configuration]

The middleware reads `token.username` to check if onboarding is complete. The JWT callback in `src/lib/auth.ts` must include `username` in the token. Update the `jwt` callback in `authOptions`:

```typescript
async jwt({ token, account, trigger, session }) {
  if (account) token.sub = account.providerAccountId;

  // On session update or first sign-in, fetch fresh user data
  if (trigger === "update" || account) {
    const dbUser = await db.query.users.findFirst({
      where: eq(users.googleId, token.sub as string),
    });
    if (dbUser) {
      token.username = dbUser.username;
      token.userId = dbUser.id;
      token.walkthroughSeen = dbUser.walkthroughSeen;
    }
  }

  return token;
},
```

And extend `src/types/next-auth.d.ts`:
```typescript
declare module "next-auth/jwt" {
  interface JWT {
    username?: string;
    userId?: string;
    walkthroughSeen?: boolean;
  }
}
```

### Banned User Handling

[Source: prd.md#8.10-admin-dashboard]

Banned users should not be able to use the app. Add a check in the `signIn` callback:

```typescript
async signIn({ account }) {
  if (account?.provider === "google") {
    const existingUser = await db.query.users.findFirst({
      where: eq(users.googleId, account.providerAccountId),
    });

    if (existingUser?.status === "banned") {
      return "/login?error=AccountBanned";
    }

    // Upsert new user (existing logic)
    await db.insert(users)
      .values({ googleId: account.providerAccountId, name: user.name!, username: "" })
      .onConflictDoNothing();
  }
  return true;
},
```

Create a simple banned user error display on the login page: when `?error=AccountBanned` is in the URL, show: "Your account has been permanently suspended."

### `callbackUrl` Handling

[Source: architecture.md#10-frontend-architecture]

NextAuth handles `callbackUrl` automatically when `signIn()` is called from a protected page. The user is redirected to the originally requested URL after successful login. No custom implementation needed ‚Äî this is built into NextAuth's redirect behaviour.
