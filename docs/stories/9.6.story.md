# Story 9.6 — Delete Post and Comment

**Epic:** 09 — Community
**Story:** 9.6
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 9.3 (post detail with kebab menu, CommentThread component)

---

## Story

**As a** post or comment author,
**I want** to delete my own posts and comments,
**so that** I can remove content I no longer want to be visible.

---

## Acceptance Criteria

1. - [ ] "Delete" option in post detail kebab menu — own posts only; 403 if another user calls the API
2. - [ ] Delete post: shows `ConfirmModal`; on confirm: `DELETE /api/posts/[id]`; cascades comments, reactions, reports in DB
3. - [ ] Post images deleted from R2 on post deletion
4. - [ ] After deletion: navigate back to `/community` feed
5. - [ ] Delete comment: own comments only; `ConfirmModal`; `DELETE /api/comments/[id]`
6. - [ ] If deleted comment has replies: replace `content` with `"[deleted]"` and set `deleted=true` (soft delete); keep record; replies still visible
7. - [ ] If deleted comment has NO replies: hard delete the record
8. - [ ] `communityService.deletePost(postId, userId)` — ownership check + R2 cleanup
9. - [ ] `communityService.deleteComment(commentId, userId)` — ownership check; soft or hard delete depending on replies
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

### communityService.deletePost()

[Source: architecture.md#6-components]

Add to `src/services/communityService.ts`:

```typescript
async deletePost(postId: string, userId: string): Promise<void> {
  const post = await db.query.posts.findFirst({
    where: eq(posts.id, postId),
  });
  if (!post) throw new NotFoundError("Post not found");
  if (post.userId !== userId) throw new ForbiddenError("You do not own this post");

  // Delete R2 images
  if (post.imageUrls && post.imageUrls.length > 0) {
    await Promise.allSettled(
      post.imageUrls.map(key =>
        deleteObject(key).catch(err =>
          logger.warn({ err, key }, "R2 delete failed for post image")
        )
      )
    );
  }

  // Delete post (cascades reactions, comments, reports in DB)
  await db.delete(posts).where(eq(posts.id, postId));
}
```

The `ON DELETE CASCADE` on `post_reactions`, `post_comments`, and `post_reports` foreign keys handles the DB cleanup automatically.

### communityService.deleteComment()

[Source: architecture.md#6-components]

Add to `src/services/communityService.ts`:

```typescript
async deleteComment(commentId: string, userId: string): Promise<void> {
  const comment = await db.query.postComments.findFirst({
    where: eq(postComments.id, commentId),
    with: {
      replies: { columns: { id: true } },
    },
  });
  if (!comment) throw new NotFoundError("Comment not found");
  if (comment.userId !== userId) throw new ForbiddenError("You do not own this comment");

  const hasReplies = comment.replies && comment.replies.length > 0;

  if (hasReplies) {
    // Soft delete: replace content, mark deleted
    await db.update(postComments)
      .set({ content: "[deleted]", deleted: true })
      .where(eq(postComments.id, commentId));
  } else {
    // Hard delete
    await db.delete(postComments).where(eq(postComments.id, commentId));
  }
}
```

export async function DELETE(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    await communityService.deletePost(params.id, session.user.id);
    return NextResponse.json({ ok: true });
  } catch (error) {
    return handleApiError(error);
  }
}
