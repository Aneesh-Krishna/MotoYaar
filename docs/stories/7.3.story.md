**Status:** Draft

## Story

**As a** user,
**I want** to trigger an AI-generated narrative analysis of my spending,
**so that** I receive personalised insights without waiting for the generation to complete synchronously.

---

## Acceptance Criteria

1. - [ ] "Generate AI Report" button on Overall Spend Report page shows remaining free quota: e.g. "1 free report available this month"
2. - [ ] If no expense data exists for the selected period: button is **disabled** and shows "No data to analyse"; free quota is NOT consumed
3. - [ ] On click (with data present): `POST /api/reports/ai` creates an `ai_reports` record with `status='pending'`, triggers async generation, and returns `{ reportId }`
4. - [ ] User immediately sees a toast: "Your report is being generated. We'll notify you when it's ready." — no blocking wait
5. - [ ] After the free report is consumed for the month: button is replaced with "Come back next month for your next free report" (static text, no button)
6. - [ ] Free report counter logic: `COUNT ai_reports WHERE user_id = X AND requested_at >= start_of_month AND status != 'failed'` — max 1 per month
7. - [ ] Quota exceeded: API returns 403 with error code `QUOTA_EXCEEDED`; `QuotaExceededError` used from `src/lib/errors.ts`
8. - [ ] Async generation: after returning `{ reportId }`, generation proceeds outside the HTTP response using Vercel's streaming/fire-and-forget pattern OR client polls `/api/reports/ai/[id]/generate`; see Dev Notes
9. - [ ] `aiReportService.requestReport(userId, periodLabel, expenseData)` in `src/services/aiReportService.ts` — creates the DB record and triggers generation
10. - [ ] `generateReport()` from `src/lib/anthropic.ts` called with aggregated expense data (not raw rows)
11. - [ ] On successful generation: `ai_report.status = 'ready'`, `ai_report.content = narrative`; notification triggered (Story 7.5)
12. - [ ] On generation failure: `ai_report.status = 'failed'`; quota is NOT counted for failed reports
13. - [ ] `pnpm typecheck` and `pnpm lint` pass

```sql
CREATE TABLE ai_reports (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  status      TEXT NOT NULL DEFAULT 'pending',  -- 'pending' | 'ready' | 'failed'

  completed_at TIMESTAMPTZ,
  expense_snapshot JSONB NOT NULL               -- aggregated data sent to Claude
);
```

### Free Quota Check

[Source: architecture.md#5-api-specification]

```typescript
// In aiReportService.ts
async checkQuota(userId: string): Promise<{ allowed: boolean; usedThisMonth: number }> {
  const startOfMonth = new Date();
  startOfMonth.setDate(1);
  startOfMonth.setHours(0, 0, 0, 0);

  const count = await db.query.aiReports.findMany({
    where: and(
      eq(aiReports.userId, userId),
      gte(aiReports.requestedAt, startOfMonth),
      ne(aiReports.status, "failed"),
    ),
  });

  return { allowed: count.length < 1, usedThisMonth: count.length };
}
```

The monthly limit is **1 free report** for MVP. The constant `FREE_REPORTS_PER_MONTH = 1` should be defined at the top of `aiReportService.ts` so it is easy to change.

```typescript
export async function POST(req: Request) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

    const body = await req.json();
    // body: { periodLabel, expenseSnapshot } — assembled by client from current report state

      body.periodLabel,
      body.expenseSnapshot

}
```

### aiReportService.requestReport()

[Source: architecture.md#6-components]

Create `src/services/aiReportService.ts`:

```typescript
async requestReport(
  userId: string,
  periodLabel: string,
  expenseSnapshot: ExpenseSnapshot
): Promise<{ reportId: string }> {
  // 1. Quota check
  const { allowed } = await this.checkQuota(userId);
  if (!allowed) throw new QuotaExceededError("AI report quota exceeded for this month");

  // 2. Validate data
  if (!expenseSnapshot || expenseSnapshot.totalExpenses === 0) {
    throw new ValidationError("No expense data to analyse");
  }

  // 3. Create pending record
  const [record] = await db.insert(aiReports).values({
    userId,
    periodLabel,
    status: "pending",
    expenseSnapshot,
  }).returning();

  // 4. Trigger async generation (fire-and-forget)
  this.generateAsync(record.id, userId, expenseSnapshot, periodLabel);
  // Note: do NOT await — returns immediately

  return { reportId: record.id };
}
```

### Async Generation Pattern

[Source: architecture.md#10-background-jobs]

**Recommended approach for Vercel**: Use the client-side "fire and trigger" pattern — after receiving `{ reportId }` from `POST /api/reports/ai`, the client immediately calls `POST /api/reports/ai/[id]/generate` in a separate non-awaited fetch (the client does not wait for a response). This second endpoint does the heavy lifting:

```typescript
// Client side (after receiving reportId)
fetch(`/api/reports/ai/${reportId}/generate`, { method: "POST" })
  .catch(() => {}); // intentionally ignored
```

Create `src/app/api/reports/ai/[id]/generate/route.ts`:

```typescript
export async function POST(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    await aiReportService.runGeneration(params.id, session.user.id);
    return NextResponse.json({ ok: true });
  } catch (error) {
    return handleApiError(error);
  }
}
```

```typescript
// aiReportService.runGeneration()
async runGeneration(reportId: string, userId: string): Promise<void> {
  const report = await db.query.aiReports.findFirst({
    where: and(eq(aiReports.id, reportId), eq(aiReports.userId, userId)),
  });
  if (!report || report.status !== "pending") return;

  try {
    const content = await generateReport(report.expenseSnapshot);
    await db.update(aiReports)
      .set({ status: "ready", content, completedAt: new Date() })
      .where(eq(aiReports.id, reportId));

    // Trigger notification (Story 7.5)
    await notificationService.createAIReportReadyNotification(userId, reportId, report.periodLabel);
  } catch (err) {
    logger.error({ err, reportId }, "AI report generation failed");
    await db.update(aiReports)
      .set({ status: "failed" })
      .where(eq(aiReports.id, reportId));
  }
}
```

Alternative: If Vercel background functions are available (Pro plan), use `waitUntil` from `@vercel/functions`. Include a note in the file for future migration.

```typescript
interface ExpenseSnapshot {

  currency: string;

  monthlyTotals: Array<{ month: string; total: number }>;
  vehicleCount: number;
  topVehicle?: { name: string; total: number };

```

The service assembles this from the already-computed report data (same data as shown on screen) before sending to the API.

### Error Handling

[Source: architecture.md#7-error-handling]

- `QuotaExceededError` → 403 `QUOTA_EXCEEDED`
- No expense data → 400 validation error (button disabled client-side, defence in depth)
- Generation failure → status set to `failed`; no user-visible error at time of request (they get a notification when ready, or lack of notification informs them it failed — MVP acceptable)
