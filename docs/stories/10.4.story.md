# Story 10.4 — Manage Access (Owner: View & Revoke)

**Epic:** 10 — Vehicle Sharing
**Story:** 10.4
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 10.1 (vehicle_invites and vehicle_access tables + sharing page created)

## Story

**As a** vehicle owner,
**I want** to view who has access to my vehicle and revoke that access if needed,
**so that** I remain in full control of who can see my vehicle data.

### vehicleInviteService Methods

[Source: architecture.md#6-components]

Add to `src/services/vehicleInviteService.ts`:

```typescript
async listAccess(
  vehicleId: string,
  ownerUserId: string
): Promise<{ access: VehicleAccessEntry[]; pendingInvites: VehicleInvite[] }> {
  // Ownership check
  await vehicleService.getWithAccessCheck(vehicleId, ownerUserId);

  const [access, pendingInvites] = await Promise.all([
    db.query.vehicleAccess.findMany({
      where: eq(vehicleAccess.vehicleId, vehicleId),
      with: { user: { columns: { id: true, name: true, email: true, image: true } } },
    }),
    db.query.vehicleInvites.findMany({
      where: and(
        eq(vehicleInvites.vehicleId, vehicleId),
        eq(vehicleInvites.status, "pending"),
        gte(vehicleInvites.expiresAt, new Date()),
      ),
    }),
  ]);

  return { access, pendingInvites };
}

async revokeAccess(vehicleAccessId: string, ownerUserId: string): Promise<{ vehicleName: string; revokedUserId: string }> {
  // Fetch the access record including vehicle to verify ownership
  const accessRecord = await db.query.vehicleAccess.findFirst({
    where: eq(vehicleAccess.id, vehicleAccessId),
    with: { vehicle: true },
  });
  if (!accessRecord) throw new NotFoundError("Access record not found");
  if (accessRecord.vehicle.userId !== ownerUserId) {
    throw new ForbiddenError("You do not own this vehicle");
  }

  await db.delete(vehicleAccess).where(eq(vehicleAccess.id, vehicleAccessId));

  return {
    vehicleName: accessRecord.vehicle.name,
    revokedUserId: accessRecord.userId,
  };
}

async cancelInvite(inviteId: string, ownerUserId: string): Promise<void> {
  const invite = await db.query.vehicleInvites.findFirst({
    where: eq(vehicleInvites.id, inviteId),
  });
  if (!invite) throw new NotFoundError("Invite not found");
  if (invite.ownerUserId !== ownerUserId) throw new ForbiddenError("Not your invite");

  await db.delete(vehicleInvites).where(eq(vehicleInvites.id, inviteId));
}
```

```typescript
  }

```

Create `src/app/api/invites/access/[accessId]/route.ts`:

```typescript
export async function DELETE(req: Request, { params }: { params: { accessId: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    const result = await vehicleInviteService.revokeAccess(params.accessId, session.user.id);
    // Send revocation notification
    await notificationService.create(
      result.revokedUserId,
      "access_revoked",
      "Your access has been removed",
      `Your access to ${result.vehicleName} has been removed by the owner.`,
      "/garage"
    );
    return NextResponse.json({ ok: true });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### Expired Invites Cleanup

[Source: architecture.md#3-database-schema]

Expired invites (status='pending' but `expires_at < now()`) are not shown in the pending list (filtered in `listAccess()`). The cron job or a lazy cleanup can mark them as 'expired'. For MVP, simply exclude them from display via the query filter.
