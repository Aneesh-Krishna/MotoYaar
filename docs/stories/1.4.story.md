### Resend Account Setup

[Source: architecture.md#2-high-level-architecture]

1. Create a Resend account at resend.com (free: 3,000 emails/month)
2. Add and verify a sending domain (e.g. `motoyaar.app`) via DNS records
3. Create an API key → store as `RESEND_API_KEY`
4. The `from` address for all emails: `MotoYaar <noreply@motoyaar.app>` (or your verified domain)

> For local development, Resend allows sending to your own email without a custom domain — use `onboarding@resend.dev` as the from address during development only.

### Resend Client Implementation (`src/lib/resend.ts`)

[Source: architecture.md#12-unified-project-structure]

```typescript
import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY!);

const FROM_ADDRESS = "MotoYaar <noreply@motoyaar.app>";

/** Generic email sender. All emails go through this function. */
export async function sendEmail(
  to: string,
  subject: string,
  html: string
): Promise<void> {
  const { error } = await resend.emails.send({
    from: FROM_ADDRESS,
    to,
    subject,
    html,
  });

  if (error) {
    throw new Error(`Resend error: ${error.message}`);
  }
}
```

> **Server-side only.** This function must only be called from API route handlers or server-side code. Never import in client components.
