### R2 Client Implementation (`src/lib/r2.ts`)

[Source: architecture.md#12-unified-project-structure, architecture.md#3-tech-stack]

Implement `src/lib/r2.ts` using AWS SDK v3 (S3-compatible API):

```typescript
import { S3Client, PutObjectCommand, DeleteObjectCommand, GetObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const r2 = new S3Client({
  region: "auto",
  endpoint: `https://${process.env.R2_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID!,
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
  },
});

const BUCKET = process.env.R2_BUCKET_NAME!;

/** Generate a pre-signed PUT URL for client-side upload (expires in 60s) */
export async function generateUploadUrl(key: string, contentType: string): Promise<string> {
  const command = new PutObjectCommand({
    Bucket: BUCKET,
    Key: key,
    ContentType: contentType,
  });
  return getSignedUrl(r2, command, { expiresIn: 60 });
}

/** Generate a pre-signed GET URL for private document access (15-min TTL) */
export async function generateAccessUrl(key: string): Promise<string> {
  const command = new GetObjectCommand({ Bucket: BUCKET, Key: key });
  return getSignedUrl(r2, command, { expiresIn: 900 }); // 15 minutes
}

/** Delete an object from R2 */
export async function deleteObject(key: string): Promise<void> {
  await r2.send(new DeleteObjectCommand({ Bucket: BUCKET, Key: key }));
}
```

**Key naming convention:** Use `{userId}/{documentId}/{filename}` pattern for documents and `{userId}/receipts/{expenseId}/{filename}` for receipts.

### Security: Signed URLs

[Source: architecture.md#15-security-and-performance]

- All document URLs are **pre-signed with a 15-minute TTL** — never stored as public URLs
- Never expose signed URLs in client-side state longer than needed
- Signed URLs must never be cached by the service worker (handled in Story 1.6)
- The R2 bucket should NOT have public access enabled

---

### Lifecycle Rule: Auto-Delete 10 Days Post-Expiry

[Source: prd.md#8.3-document-management]

Documents in full-storage mode must be auto-deleted 10 days after the document's expiry date.

R2 does not natively support tag-based lifecycle rules in the same way as AWS S3. Two approaches:

**Option A (Recommended for MVP):** Handle deletion in the daily cron job (Story 8.1). The cron already processes documents — add a step to delete R2 objects where `expiry_date < today - 10 days AND storage_url IS NOT NULL`. This is simpler and more reliable at MVP scale.

**Option B:** Use R2's built-in lifecycle rules with object expiry headers set at upload time. Set `Expires` header to `expiry_date + 10 days` when uploading.

> Implement Option A (cron-based deletion). Leave a code comment referencing Story 8.1 for the deletion logic.
