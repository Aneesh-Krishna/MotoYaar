# Story 5.1 — Add Expense

**Epic:** 05 — Expense Tracking
**Story:** 5.1
**Status:** Draft
**Created:** 2026-03-15
**Depends On:** Story 3.4 (Vehicle Detail Page — Add Expense CTA lives on Expenses tab)

---

## Story

**As a** vehicle owner,
**I want** to log an expense for my vehicle,
**so that** I have a record of every cost associated with it.

---

## Acceptance Criteria

1. - [ ] Expense form opens in a `BottomSheet` on mobile (same BottomSheet from Epic 04)
2. - [ ] Fields: Price (required, numeric, `₹` prefix), Date (required, date picker, defaults to today), Reason (required, chip selector: Service / Fuel / Trip / Others), Where (optional text), Comments (optional dropdown: Overpriced / Average / Underpriced), Receipt upload (optional, Story 5.5 implements the R2 logic — stub receipt field here)
3. - [ ] If Reason = "Trip" is selected: dismiss the expense sheet and open the Add Trip flow (Story 6.1); do NOT create an expense directly
4. - [ ] Past date warning: if selected date is more than 1 year before today, show inline warning "This date seems far in the past — double-check?" but allow saving
5. - [ ] `POST /api/vehicles/[id]/expenses` (when opened from vehicle detail) or `POST /api/expenses` (general, vehicle-less)
6. - [ ] `expenseService.create(userId, vehicleId?, data)` called by the route handler
7. - [ ] Zod schema: `createExpenseSchema` in `src/lib/validations/expense.ts`
8. - [ ] Vehicle context is pre-filled (and uneditable) when the form is opened from a vehicle detail page
9. - [ ] On save: expense record created; BottomSheet closed; parent component refreshes via `router.refresh()`
10. - [ ] `pnpm typecheck` and `pnpm lint` pass

### Reason Chip Selector

[Source: front-end-spec.md#5-component-library--ui-patterns]

```tsx
const REASONS: ExpenseReason[] = ["Service", "Fuel", "Trip", "Others"];

```

When "Trip" is tapped: do NOT submit the form. Instead, call `onTripRedirect()` which dismisses this BottomSheet and opens the Trip form (Story 6.1). The parent component handles this by managing both sheets' open states.

### API Routes

[Source: architecture.md#5-api-specification]

```typescript
// src/app/api/vehicles/[id]/expenses/route.ts
export async function POST(req: Request, { params }: { params: { id: string } }) {
  const session = await getServerSession(authOptions);
  if (!session) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });

  try {
    await vehicleService.getWithAccessCheck(params.id, session.user.id);
    const body = await req.json();
    const data = createExpenseSchema.parse(body);
    const expense = await expenseService.create(session.user.id, params.id, data);
    return NextResponse.json(expense, { status: 201 });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### expenseService.create()

[Source: architecture.md#6-components]

Add to `src/services/expenseService.ts`:

```typescript
async create(userId: string, vehicleId: string | undefined, data: CreateExpenseInput): Promise<Expense> {
  const [expense] = await db.insert(expenses).values({
    userId,
    vehicleId: vehicleId ?? null,
    price: data.price,
    currency: "INR",   // user currency — defaults to INR in MVP
    date: new Date(data.date),
    reason: data.reason,
    whereText: data.whereText ?? null,
    comment: data.comment ?? null,
    receiptUrl: null,   // populated in Story 5.5
  }).returning();

  return expense;
}
```

### Zod Schema

[Source: architecture.md#17-coding-standards]

Create `src/lib/validations/expense.ts`:

```typescript
import { z } from "zod";

export const createExpenseSchema = z.object({
  price: z.number().positive("Price must be greater than 0"),
  date: z.string().min(1, "Date is required"),
  reason: z.enum(["Service", "Fuel", "Trip", "Others"]),
  whereText: z.string().optional(),
  comment: z.enum(["Overpriced", "Average", "Underpriced"]).optional(),
});

export const updateExpenseSchema = createExpenseSchema.partial();

export type CreateExpenseInput = z.infer<typeof createExpenseSchema>;
export type UpdateExpenseInput = z.infer<typeof updateExpenseSchema>;
```

### Frontend API Wrapper

[Source: architecture.md#12-unified-project-structure]

Add to `src/services/api/expenseApi.ts`:

```typescript
import { apiRequest } from "@/lib/api-client";
import type { Expense } from "@/types";
import type { CreateExpenseInput } from "@/lib/validations/expense";

export async function createVehicleExpense(vehicleId: string, data: CreateExpenseInput): Promise<Expense> {
  return apiRequest<Expense>(`/vehicles/${vehicleId}/expenses`, {
    method: "POST",
    body: JSON.stringify(data),
  });
}

export async function createExpense(data: CreateExpenseInput): Promise<Expense> {
  return apiRequest<Expense>("/expenses", {
    method: "POST",
    body: JSON.stringify(data),
  });
}
```
