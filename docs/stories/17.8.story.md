## Story

**As a** rider navigating offline,
**I want** to hear voice turn announcements from pre-downloaded audio clips and see a clear navigation UI,
**so that** I can ride safely with my phone mounted without looking at the screen for every turn.

## Dev Notes

### Voice Clip Playback from OPFS

Voice clips are stored in OPFS alongside the route graph during download (Story 17.5/17.6). Each clip is keyed as `{routeId}/voice/{stepIndex}_{type}.mp3` where `type` is `500m`, `200m`, or `at`.

```typescript
// src/lib/offline/voiceGuidance.ts
export class VoiceGuidance {
  private muted: boolean;
  private activeSource: AudioBufferSourceNode | null = null;
  private ctx: AudioContext;

  constructor() {
    this.muted = localStorage.getItem('voiceMuted') === 'true';
    this.ctx = new AudioContext();
  }

  async announce(routeId: string, stepIndex: number, type: '500m' | '200m' | 'at'): Promise<void> {
    if (this.muted) return;
    const key = `${routeId}/voice/${stepIndex}_${type}.mp3`;
    const mp3 = await readFromOPFS(key);
    if (!mp3) return; // clip not found — silent fallback
    const decoded = await this.ctx.decodeAudioData(mp3);
    const source = this.ctx.createBufferSource();
    source.buffer = decoded;
    source.connect(this.ctx.destination);
    source.start();
    this.activeSource = source;
  }

  setMuted(muted: boolean): void {
    this.muted = muted;
    localStorage.setItem('voiceMuted', String(muted));
    if (muted) this.activeSource?.stop();
  }
}
```

**Announcement thresholds:** Trigger at 500m, 200m, and at-turn based on `distanceToNextTurn` from the navigation engine. Debounce: once a clip is played for a step+type combo, do not play it again for that step.

### iOS Screen Lock — Known Limitation (Must Handle)

`SpeechSynthesis` API stops when the iOS screen locks. This is why pre-generated MP3 clips via `AudioContext` are the **primary** voice mechanism — they are not affected by screen lock in the same way.

**However**, `AudioContext` on iOS also has restrictions: it must be resumed after a user gesture. Ensure the `AudioContext` is created/resumed during a user tap (e.g., "Start Navigation" button tap).

```typescript
// Resume AudioContext on user interaction (required for iOS)
startNavigationButton.addEventListener('click', () => {
  if (voiceGuidance.ctx.state === 'suspended') {
    voiceGuidance.ctx.resume();
  }
});
```

Show a visible notice on iOS: "Screen lock will silence voice guidance — keep screen on while riding." Detect iOS via `navigator.userAgent` or feature detection.

[Source: PRD Section 8 — iOS Limitations]

### Screen Wake Lock (AC: 10)

```typescript
// src/lib/maps/wakeLock.ts
let wakeLockSentinel: WakeLockSentinel | null = null;

export async function requestWakeLock(): Promise<void> {
  if (!('wakeLock' in navigator)) return; // not supported
  try {
    wakeLockSentinel = await navigator.wakeLock.request('screen');
  } catch {
    // Permission denied or not available — silent fail
  }
}

export function releaseWakeLock(): void {
  wakeLockSentinel?.release();
  wakeLockSentinel = null;
}
```

Request on `startNavigation()`. Release on `stopNavigation()` or page unload.

[Source: PRD Section 7.2.7]

### Expiry Warning Logic

```typescript
// In OfflineRouteCard.tsx
function isExpiringSoon(expiresAt: string): boolean {
  const daysUntilExpiry = differenceInDays(new Date(expiresAt), new Date());
  return daysUntilExpiry <= 3 && daysUntilExpiry >= 0;
}

function isExpired(expiresAt: string): boolean {
  return new Date(expiresAt) < new Date();
}
```

Expired routes (> 30 days): show warning badge "Route data may be outdated" but do not block use.

[Source: PRD Section 7.3.6]

### Download Size Labelling

The size shown in the download preview comes from the `offline_routes.download_size_bytes` value estimated server-side before the actual tile download. Always label it as "~X MB" — never as an exact figure.

[Source: PRD Section 7.3.1, PRD Section 7.3.8 note on estimates]
